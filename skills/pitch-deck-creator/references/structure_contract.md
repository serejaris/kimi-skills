# Structure Contract: Business Plan Pitch Deck

This document defines the reusable section hierarchy, content patterns, and exhibit rules extracted from the reference deck (an 18-slide Chinese FP/BP for a power-electronics startup).

## CRITICAL STRUCTURE RULES

1. **Flat slide deck** — No section-divider pages. The reference flows as 18 consecutive content slides without chapter breaks.
2. **One slide, one message** — Each slide answers exactly one investor question.
3. **Actual images required** — Team page must use real headshots; product pages must use real product images. Never use generic icons or empty placeholders.
4. **Funding page uses 4 separate ring charts** — One ring per allocation category, displayed side by side.

## Document Section Hierarchy

The deck follows a classic investor-pitch flow. Each numbered item maps to exactly one slide (except Appendix which may span 2–4 slides).

| # | Section | Slide Role | Typical Slide Count |
|---|---------|-----------|---------------------|
| 1 | Cover | Title + company identity + investor branding | 1 |
| 2 | Investment Highlights | 6-point grid summary of key selling points | 1 |
| 3 | Market Opportunity | Market-size bar charts + growth thesis | 1–2 |
| 4 | Industry Landscape | Competitor mapping / peer comparison table | 1 |
| 5 | Company Positioning | Strategic pillars / value proposition bullets | 1 |
| 6 | Core Competitiveness | Advantages matrix (metrics + R&D + service) | 1–2 |
| 7 | Product Portfolio | Product overview + technical architecture | 1–2 |
| 8 | Technology Differentiation | Technical flow comparison + parameter tables | 1–2 |
| 9 | Team Introduction | Founder + core team + advisory board | 1 |
| 10 | Growth Forecast | Revenue/order projections with charts | 1 |
| 11 | Vision / Roadmap | Multi-phase strategic roadmap | 1 |
| 12 | Financing Plan | Fund-raise ask + allocation breakdown | 1 |
| 13 | Appendix | Supporting data, product photos, detailed comparisons | 3–5 |
| 14 | Disclaimer | Legal disclaimer + contact information | 1 |

**Total target**: 16–22 slides. The reference has 18 slides (13 main + 5 appendix).

**NO section divider pages** — The reference flows directly from one section to the next without chapter-title divider slides.

## Slide Layout Patterns

### Pattern A: Grid Highlights
- **Used for**: Investment Highlights (section 2).
- **Layout**: 3 columns × 2 rows = 6 feature cards filling the content area.
- **Each card**:
  - Circular icon container (thin navy blue outline) with icon inside (top).
  - Title (bold, 16–18 pt, centered).
  - Two bullet lines with key data highlighted in bold.

### Pattern B: Dual Chart
- **Used for**: Market Opportunity (section 3), Growth Forecast (section 10), Appendix market data.
- **Layout**: Two charts side by side, each with its own navy blue title bar above.
- **Title bar**: Navy blue (`#1B3A6B`) background, white bold text centered.
- **Chart style**: Navy blue bars for primary series; green bars for secondary series (if grouped).
- **Bottom**: small italic data-source line in Medium Gray.

### Pattern C: Comparison Flow
- **Used for**: Industry Landscape (section 4), Technology Differentiation (section 8).
- **Layout**: Top horizontal arrow flow (stages or categories left→right), with key company highlighted as the final arrow stage in navy blue.
- **Below**: a comparison table with rows for metrics and columns for competitors/peers.
- **Peer logos**: Display competitor logos in the flow and/or table.

### Pattern D: Strategic Pillars
- **Used for**: Company Positioning (section 5).
- **Layout**: Vertical stack of 4–5 rows. Each row = a navy blue rounded label pill (left) + bullet list (right).
- **Pill style**: Navy blue rounded rectangle (`#1B3A6B`), white bold text, approximately 160 × 40 px.
- **Bullet style**: Dark Text, regular weight, 1–2 lines per pillar.

### Pattern E: Three-Column Matrix
- **Used for**: Core Competitiveness (section 6).
- **Layout**: Upper section = three side-by-side modules separated by vertical rules. Lower section = icon grid (4 icons + labels) spanning the full width.
- **Each module**: has a bold title with a key metric highlighted, followed by 2–3 bullet points.
- **Bottom icon grid**: 4 application icons with labels (e.g., industrial vehicles, mining machinery, electric ships, energy storage).
- **Background**: White top area; a subtle Light Gray (`#F5F5F5`) bottom band for the icon grid.

### Pattern F: Top/Bottom Comparison
- **Used for**: Technology flow diagrams (section 8).
- **Layout**: Upper half = "traditional approach" (icons + sequential rectangles). Lower half = "our approach" (same structure with key components highlighted in navy blue).
- **Left callout boxes**: Dark navy rectangles on the far left showing percentage/impact (e.g., "损耗 >15%" vs "损耗 <5%").
- **Bottom banner**: Full-width navy blue rectangle with white takeaway text.

### Pattern G: Photo Grid
- **Used for**: Product showcase / Appendix product photos.
- **Layout**: 3 columns, each = caption (above) + photo (below).
- **Photo style**: Rectangular with thin shadow, consistent aspect ratio.
- **CRITICAL**: Use actual product photos, not empty placeholders. If user provides no photos, use product-renderings or skip this slide.

### Pattern H: Conclusion Banner Bottom
- **Used in**: Many slides across sections.
- **Layout**: After the main content, a full-width navy blue rectangle spans the lower 60–80 px of the slide.
- **Text**: white, 14–16 pt, centered — a one-line takeaway or key insight.

## Team Slide Layout

- **Left side**: Large founder photo + name + title below.
- **Founder bio**: 3–5 bullet points highlighting experience, achievements, awards.
- **Right side**: Advisor photo + name + title.
- **Bottom row**: 3 team member cards (name + title + 2–3 bullet bios).
- **CRITICAL**: Use actual headshot photos, not generic icons or placeholder silhouettes.

## Financing Slide Layout

- **Top**: Large centered fund-raise amount (e.g., "5,000万人民币").
- **Below**: Four separate ring/donut charts arranged horizontally, each showing one allocation:
  - Ring 1: R&D investment (e.g., 30%)
  - Ring 2: Production operations (e.g., 25%)
  - Ring 3: Capacity scale (e.g., 25%)
  - Ring 4: Market expansion (e.g., 20%)
- **Each ring**: Navy blue arc for the percentage, light gray for the remainder. Percentage label in center.
- **Labels**: Category name below each ring.
- **CRITICAL**: Use FOUR separate ring charts, NOT one combined donut chart.

## Exhibit Numbering

- Exhibits are **implicitly grouped** by section. They are labeled using the format `{Appendix|Annex}：{Descriptive Title}` (e.g., `附录：寰晟产品应用场景`).
- No global sequential numbering (Exhibit 1, 2, 3…). Instead, appendix pages have descriptive section-prefix titles.
- Charts with data are always annotated with a source: `Data source: Research Institution Name` (at bottom-left or bottom-center, 10–11 pt).

## Content Density Rules

- **One slide, one message**: each slide should answer exactly one investor question.
- **Maximum text**: 40–50 Chinese characters or ~80 English words of body text per slide.
- **Data points per slide**: 3–6 key metrics highlighted.
- **Visual-first**: always prefer a chart, diagram, or comparison table over pure text bullets.
- **Supporting evidence**: every claim should ideally have a number, a chart, or a named source.

## Cross-Reference Patterns

- Financial/projections slides are self-contained but logically ordered (past → current → projected).
- Technology comparison slides reference company advantages established in positioning slides.
- Appendix provides evidential support for claims made in the main narrative.
