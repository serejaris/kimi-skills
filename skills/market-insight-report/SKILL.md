# 市场洞察报告技能

生成以顶级咨询公司标准为范本的专业咨询风格市场洞察报告。

## 参考来源
- **类型**：上传的 PDF 素材
- **素材类型**：PDF
- **参考文件类型**：PDF
- **基于**：顶级咨询公司 + 市场研究合作伙伴联合报告（32页）
- **主要语言**：中文（简体）混合英文

## 支持的输出格式
- **PDF**（主格式，与参考素材一致）
- **DOCX**（可编辑版本）
- **PPTX**（演示版本）

## 默认输出格式选择
- 如用户明确指定格式，则使用该格式
- 否则，打印级报告默认使用 **PDF**，可编辑草稿默认使用 **DOCX**

## 参考文件

生成报告时请阅读以下文件：
- `references/analysis_framework.md` -- **分析方法论**：市场规模测算、品类分析、渠道分析、消费者行为、竞争分析、数据到洞察的方法论
- `references/chart_creation_guide.md` -- **图表渲染规范**：展示项到图表的映射关系、配色方案、9种图表类型规范、技术输出要求。**图表必须以内嵌图片形式渲染，不得使用文字占位符或表格替代。**
- `references/style_contract.md` -- 视觉风格：字体排版、配色、版面布局、图表处理
- `references/structure_contract.md` -- 文档结构：章节层级、内容流程、图表编号
- `references/sota_analysis.md` -- 参考素材的契约是如何推导出来的

## 报告生成工作流

### 第一步：理解需求
从用户查询中识别：
- **主题/领域**：哪个市场或行业？
- **时间范围**：历史回顾期 + 当前分析期
- **数据来源**：分析应基于哪些数据？
- **目标受众**：高管、营销人员、投资者？
- **核心问题**：报告需要回答哪些洞察？
- **输出格式**：PDF、DOCX 还是 PPTX？

### 第二步：开展分析
阅读 `references/analysis_framework.md` 并运用相关分析方法：
- **市场规模测算**：使用自上而下 + 自下而上交叉验证（模块1）
- **品类分析**：将增长分解为渗透率 × 购买频次 × 客单价；进行量价拆分（模块2）
- **渠道分析**：追踪渠道份额演变，构建迁移矩阵，比较渠道效率（模块3）
- **消费者行为**：描绘购买旅程，进行用户分群，分析需求场景（模块4）
- **竞争分析**：归因份额变动，计算集中度指标，构建竞争力雷达图（模块5）
- **洞察综合**：遵循5步发现流程；对每个数据点进行"So what?"检验（模块6）

报告中每一个分析论断都必须追溯到上述某个框架。

### 第三步：规划报告结构
遵循标准咨询报告架构（详见 `references/structure_contract.md`）：

```
1. 封面页（购物实景大图 + 红色标题 + 合作伙伴品牌标识）
2. 作者简介及致谢（简洁文字排版，无装饰性边框）
3. 目录
4. 摘要（1-3页密集文字，仅核心发现，无图表）
5. "完整报告"图片分隔页（大图 + 标题叠加 + 简介段落）
6. 报告正文（各章节连续排列，不设编号分隔页）
   - 章节标题用红色粗体，后接引导段落 + 含图表的分析内容
   - 各小节标题用黑色粗体
7. 战略建议 / 未来展望
8. 封底（简洁设计，含标识 + 办事处列表 + 联系方式）
```

**源自参考文件的关键规则**：
- 不设编号章节分隔页（如 01、02、03）
- 不设多余页面（无方法论说明页、无附录）
- 内容应密集排列——避免出现接近空白的页面
- 不使用面包屑式编号系统（如 1.1、1.2）

完整细节请参阅 `references/structure_contract.md`。

### 第四步：应用视觉风格

完整细节请阅读 `references/style_contract.md`。关键规则：

- **配色方案**：深红色（#CC0000）作为强调色，黑色用于正文，灰色用于图表辅助数据
- **字体排版**：阿里普惠体（中文）、Graphik（英文标题）、TiemposText（英文正文）
- **每页页眉**："合作方1 | 合作方2" 红色显示 + 水平分割线 + 副标题居中显示在下方
- **页脚**：页码居中显示在底部
- **图表**：编号格式"图X："，红色标签，每张图表下方标注数据来源

### 第五步：内容编写规则

#### 数据呈现
- 始终先给出洞察结论，再用数据支撑
- 包含具体的百分比、增长率和绝对数值
- 明确标注对比时段（如"2023年第一季度 vs 2022年第一季度"）
- 正文中使用"（见图X）"格式引用图表

#### 图表指南
- **图表必须以实际内嵌图片形式渲染**（PNG，150+ DPI），不得使用文字占位符或纯表格替代。这是第一质量要求。完整规范详见 `references/chart_creation_guide.md`。
- 主数据系列使用红色强调（`#CC0000`），辅助数据使用灰色
- 在柱形/条形上直接标注数据标签
- 每张图表下方添加数据来源标注："资料来源：..."
- 图表放置在首次文字引用附近
- 使用连续编号：图1、图2、图3...
- 图表标题应为**断言式**（"短视频占据用户时长38.4%"），而非描述式（"用户时长分布"）
- **每2个内容页至少1张图表**；理想状态为每页1-2张图表
- 9种支持的图表类型：柱形+增长线（双轴）、饼/环形图、堆叠百分比柱形图+CAGR表格、瀑布/桥接图、2×2小图网格、带标注折线图、水平条形图（排名）、雷达/蛛网图、四象限散点图
- 渲染规范、配色方案和展示项映射关系详见 `references/chart_creation_guide.md`

#### 写作风格
- **流畅的咨询叙事风格**——报告应读起来像连贯的段落，而非项目符号列表、数据表格或标签式板块（洞察/驱动因素/建议等标签属于分析脚手架，在输出中不可见）
- 每个段落遵循"论断 → 证据 → 解读 → 启示"的自然行文流程
- 专业、分析性的语调；第三人称视角
- 仅限数据支撑的论断——每个论点必须通过"So what?"检验（详见 `references/analysis_framework.md` 模块6）
- **在行文中加粗关键数据**（"用户规模达**11.16亿**"），切勿孤立列举数据
- **有对比，不孤立**——每个数字都需要一个基准参照（同比、vs 共识预期、vs 同业）
- 将清晰的因果关系解释编织进叙事中，而非作为独立板块列出
- 章节以设定背景的引导段落开篇，引出核心问题；以过渡段落结尾，衔接下一章节
- 建议应写成有论据支撑的结论，配以具体行动方案，而非项目符号式的愿望清单
- 增长分解（渗透率 × 购买频次 × 客单价）应贯穿每一次品类分析
- 段落模板、过渡技巧和示例请参阅 `references/structure_contract.md` "咨询叙事写作规则"部分

#### 内容密度
- 密集排列内容；避免出现接近空白的页面
- 每页应有充实的文字（800-1200字）或1-2张全尺寸图表
- 重点段落应与相邻内容合并，不应独占整页

### 第六步：字体处理（CJK 兼容）

- **主要中文字体**：阿里普惠体（Alibaba PuHuiTi）—— Light、Regular、Medium、Bold
- **英文标题字体**：Graphik（Light、Regular、Semibold）
- **英文正文字体**：TiemposText（Regular、RegularItalic）
- 如阿里普惠体不可用：替换为 Noto Sans SC 或思源黑体（Source Han Sans SC）
- 如 Graphik 不可用：替换为 Montserrat 或 Inter
- 如 TiemposText 不可用：替换为 Georgia 或 Noto Serif
- 所有文本路径使用一致的字体策略——不要将拉丁文默认字体与中文补丁混用

## 输出格式说明

### PDF 输出
- 严格保持视觉风格契约：基于摄影的封面、红色标题文字、合作伙伴标识
- 所有内容页面（从目录到报告正文）保持页眉，封面、作者页和封底不设页眉
- 必须嵌入 CJK 字体
- 页码从目录开始（第1页）

### DOCX 输出
- 保持相同的章节层级和标题样式
- 封面页使用摄影图片，标题文字叠加或置于图片下方
- 红色 H1 标题，黑色 H2/H3 标题
- 图表以图片或 Office 原生图表形式内嵌
- 页眉/页脚与视觉风格契约保持一致

### PPTX 输出
- 将报告转换为幻灯片格式：每个主要章节1-2页幻灯片
- 封面幻灯片使用主视觉图片 + 标题
- 内容幻灯片：洞察标题 + 要点列表 + 关键图表
- 保持红/灰/黑配色方案
