# 图表创建指南 — 市场洞察报告

## 强制要求：图表必须以内嵌图片形式渲染

**每一个数据可视化展示项都必须包含实际渲染的图表图片，不得使用文字描述、占位符或纯表格替代。** 图表是报告论据的主要视觉证据。仅含表格而无图表图片的报告不达标。

在生成输出文件（PDF/DOCX）时，使用 matplotlib、plotly 或同等图表库渲染图表图片并内嵌到文档中。表格是图表的补充——不能替代图表。

## 展示项到图表的映射关系

使用此映射关系判断哪些展示项需要图表图片，哪些使用表格：

| 分析章节 | 展示项类型 | 是否需要图表 | 推荐图表类型 |
|-----------------|-------------|-------------|----------------------|
| 市场规模测算 / TAM | 市场规模趋势 | **是** | 柱形图 + 增长率折线（双轴） |
| 市场规模测算 / 交叉验证 | 对比表格 | 否——仅表格 | — |
| 增长分解 | 驱动因素对比 | **是** | 分组柱形图或瀑布图 |
| 时间序列趋势（用户、时长） | 趋势线 | **是** | 带标注折线图 |
| 品类/细分份额 | 份额构成 | **是** | 饼/环形图或堆叠百分比柱形图 |
| 渠道份额演变 | 跨时段份额 | **是** | 堆叠百分比柱形图 + CAGR 侧表 |
| 渠道迁移 | 迁移矩阵 | 否——仅表格 | — |
| 渠道效率 | 效率对比 | **是** | 分组柱形图 |
| 消费者分群 | 人群对比 | **是** | 雷达/蛛网图或分组柱形图 |
| 消费者需求场景 | 象限映射 | **是** | 四象限散点图 |
| 竞争格局 | 市场份额 | **是** | 饼/环形图或水平条形图 |
| 竞争归因 | 份额桥接 | **是** | 瀑布/桥接图 |
| 平台排名 | App MAU 排名 | **是** | 水平条形图 |
| 预测/展望 | 预测趋势 | **是** | 折线图（含预测分界线） |
| 详细财务数据 | 数据表格 | 否——仅表格 | — |

**经验法则**：如果数据有视觉故事可讲（趋势、对比、分布），使用图表。如果数据是查阅型/参考型（详细数字、多列对比），使用表格。大多数章节应**至少包含一张图表图片**。

## 图表配色方案

所有图表必须使用此配色方案，源自报告的咨询视觉风格：

| 角色 | 颜色 | 色值 | 用途 |
|------|-------|-----|-------|
| 主色强调 | 深红色 | `#CC0000` | 最近期数据、关键系列、高亮品类 |
| 辅助色 | 深灰色 | `#666666` | 次要数据系列、前一时段 |
| 第三色 | 中灰色 | `#999999` | 较早数据、背景系列 |
| 正向 | 青绿色 | `#2E8B8B` | 正向变化、增长 |
| 负向 | 珊瑚红 | `#E74C3C` | 负向变化、下降 |
| 背景 | 浅灰色 | `#F5F5F5` | 图表背景填充 |
| 网格线 | 浅灰色 | `#CCCCCC` | 仅水平网格线 |
| 多系列轮换 | — | `#CC0000`、`#666666`、`#2E8B8B`、`#E8913A`、`#4A90D9`、`#999999` | 用于3个以上数据系列的图表 |

**关键要求**：深红色（`#CC0000`）始终作为图表的主色（最重要的数据系列）。切勿使用鲜艳的彩虹色配色方案。

## 图表类型规范

### 1. 柱形图 + 增长折线（双轴）

**适用场景**：市场规模趋势、营收趋势、用户增长的时间序列

**布局**：
- 柱形对应左侧主 Y 轴：绝对值（营收、用户数）
- 折线对应右侧副 Y 轴：增长率（%）
- X 轴：时间区间（年度或季度）

**渲染规范**：
- 柱形填充：深红色（`#CC0000`）实心，宽度约0.6
- 折线：深灰色（`#666666`），2px 实线，圆形标记
- 背景：白色，无边框
- 仅水平网格线，浅灰色（`#CCCCCC`），0.5px
- 无垂直网格线
- 柱形上方标注数据标签（数值），折线点上标注百分比
- Y 轴标签带单位（"亿元"、"%"）
- 图例：右上角或正上方居中，位于图表区域内

**长宽比**：16:9（约宽14cm，高8cm）

### 2. 饼/环形图

**适用场景**：市场份额、营收结构、渠道占比、用户构成

**布局**：
- 环形图优先于饼图（内径约为外径的55%）
- 最多6个区段；较小的区段合并为"其他"
- 最大区段从12点钟方向开始，顺时针递减

**渲染规范**：
- 主区段：深红色（`#CC0000`）
- 其他区段：依次使用 `#666666`、`#2E8B8B`、`#E8913A`、`#4A90D9`、`#999999`
- 区段之间使用白色分隔线（1px）
- 百分比标签位于区段内部或外部（粗体，10pt）
- 品类标签在外部配引导线，或在下方图例中标注
- 无3D效果，无投影

### 3. 堆叠百分比柱形图 + CAGR 侧表

**适用场景**：渠道份额演变、品类份额时间趋势、营收结构变化

**布局**：
- 堆叠柱形高度为100%，每个时间区间一个柱形
- X 轴：时间区间
- Y 轴：0%–100%
- 右侧表格：各区段在时间跨度内的 CAGR

**渲染规范**：
- 各区段使用多系列色彩轮换
- 百分比标签位于每个区段内部（深色背景用白色文字，浅色背景用黑色文字）
- CAGR 侧表：小型，与柱形图对齐，纯文字配粗体数值
- 在25%、50%、75%处设细水平网格线

### 4. 瀑布/桥接图

**适用场景**：市场份额变化归因、营收桥接、增长分解

**布局**：
- 水平条形以零点为中心
- 绿色（#2E8B8B）表示正向贡献（增长）
- 红色（#E74C3C）表示负向贡献（下降）
- 深灰色表示合计/起始/终止条形
- 条形之间有连接线（浅灰色虚线）

**渲染规范**：
- 品类标签位于 Y 轴左侧对齐
- 数值标签位于每个条形末端
- 零线：深灰色实线，1px
- 起始条形（左）："起始份额"
- 结束条形（右）："最终份额"

### 5. 多面板小图（2×2 网格）

**适用场景**：品类对比、细分市场拆解、区域对比

**布局**：
- 2列 × 2行 = 一个展示项中4张小图
- 每个面板：独立标题，相同图表类型，共享 Y 轴刻度
- 面板之间小间隔（3-5mm）

**渲染规范**：
- 每个面板约7cm × 5cm
- 4个面板共享 Y 轴范围以保证可比性
- 面板标题：粗体，9pt，位于每个小图上方
- 简化坐标轴：减少刻度标签，不重复轴标题
- 突出值使用深红色，其他使用灰色

### 6. 带标注折线图

**适用场景**：含事件标记的时间序列趋势（政策变化、产品发布、市场事件）

**布局**：
- 折线图含1-4个数据系列的时间序列
- 垂直标注条或箭头标记关键事件
- 事件标签在小文字框中

**渲染规范**：
- 主系列：深红色（`#CC0000`），2px 实线
- 次要系列：`#666666`，1.5px 实线
- 事件标注：垂直灰色虚线 + 圆角文字框中的标签（8pt）
- 预测分界：垂直虚线 + "预测"标签配右指箭头
- 图例：位于图表区域内顶部

### 7. 水平条形图（排名）

**适用场景**：App 排名、平台对比、因素重要性排名

**布局**：
- 水平条形，按数值排序（最大值在顶部）
- 品类标签位于 Y 轴
- 数值标签位于每个条形末端

**渲染规范**：
- 条形填充：第一名使用深红色（`#CC0000`），往下渐变为灰色
- 或：所有条形使用深红色，关键对比项高亮处理
- 数值标签：粗体，位于条形末端右侧
- 无垂直网格线；可选浅色水平网格线

### 8. 雷达/蛛网图

**适用场景**：多维竞争对比、品牌竞争力评估

**布局**：
- 5-6条轴线从中心辐射
- 2-4个重叠多边形（每个竞品/细分对应一个）
- 刻度：中心为0，外缘为最大值

**渲染规范**：
- 每个竞品使用配色方案中的不同颜色
- 填充：半透明（alpha 约0.15）
- 线条：实线，2px
- 轴标签：9pt，位于每条辐射线外端
- 网格：同心五/六边形，浅灰色（#CCCCCC）
- 图例：位于图表下方

### 9. 四象限散点图

**适用场景**：品类定位、市场机会映射、需求场景分析

**布局**：
- X 轴和 Y 轴代表不同指标
- 象限分割线：在轴中位数处的虚线
- 每个点标注品类/品牌名称
- 点的大小可编码第三个变量（如市场规模）

**渲染规范**：
- 点：按组别着色（使用多系列配色方案）
- 标签：8pt，避免重叠
- 象限标签：粗体，10pt，位于每个角落
- 轴标题带单位
- 背景：白色，象限背景可有极浅的色调区分

## 图表标题与来源标注区块

每张图表必须包含：

```
图 N：[描述性断言式标题——图表展示了什么洞察，而非仅描述数据]
                    ← 图表图片 →
资料来源：[数据来源]；[公司名]分析
```

- "图 N" 使用**深红色**，粗体，10pt
- 描述使用**黑色**，粗体，10pt
- 来源行：灰色，斜体，8pt，位于图表下方
- 可选备注行：位于图表与来源之间，灰色，8pt

**标题撰写规则**：图表标题应为**断言式**，而非描述式。
- 正确："图3：短视频占据用户时长38.4%，抖音一家占半壁江山"
- 错误："图3：用户时长分布"

## 图表密度规则

- **最低要求**：每2个内容页1张图表
- **理想状态**：每个内容页1-2张图表
- **最大限度**：每页3张图表（仅在附录/对比章节中使用）
- **摘要部分**：不含图表——纯文字
- 如某章节有3段以上分析文字却没有图表，应添加一张

## 输出技术要求

- **分辨率**：最低150 DPI，推荐200 DPI
- **格式**：PNG（确保在 PDF/DOCX 中一致渲染）
- **内嵌方式**：图表必须作为行内图片嵌入文档流中，紧跟首次文字引用之后
- **尺寸**：标准图表约14cm宽 × 8cm高；双面板约14cm × 每面板6cm；小图约每面板7cm × 5cm
- **图表内字体**：使用系统无衬线字体（苹方 SC、Noto Sans SC、Arial），与报告正文字体匹配
- **抗锯齿**：开启以确保线条平滑渲染
