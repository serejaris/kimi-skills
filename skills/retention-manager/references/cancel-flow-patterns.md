# 取消流程模式

按业务类型、计费服务商和行业分类的详细取消流程模式。

---

## 按业务类型分类

### B2C / 自助服务 SaaS

高频、低接触。流程必须无需人工干预即可运转。

**流程结构：**
```
取消按钮 → 退出调查（1 个问题） → 动态方案 → 确认 → 取消后跟进
```

**特点：**
- 全自动，无需人工介入
- 快速——最多 2-3 个页面
- 一个方案 + 一个备选，不是选项菜单
- 移动端优化（大量取消发生在手机上）
- 每一步都有清晰的"继续取消"选项

**典型挽留率：** 20-30%

**示例流程（¥29/月的效率工具）：**
1. "你取消的主要原因是什么？" → 6 个选项
2. 选择了"太贵" → "接下来 3 个月享受 75 折（节省 ¥21.75）"
3. 拒绝 → "或者切换到我们的入门版 ¥12/月"
4. 拒绝 → "我们很遗憾看到你离开。你的访问权限持续到 [日期]。"

---

### B2B / 团队套餐

低频、高价值。人工跟进值得投入。

**流程结构：**
```
取消按钮 → 退出调查 → 方案（或转交客户成功团队） → 确认 → 取消后跟进
```

**特点：**
- MRR 超过阈值的账户转交客户成功团队
- 展示团队影响（"你的 8 名团队成员将失去访问权限"）
- 为企业客户提供管理员之间的电话沟通
- 更长的考虑时间——允许"预约电话"作为挽留选项
- 要求管理员/所有者身份才能取消（不是任意团队成员）

**典型挽留率：** 30-45%（因人工介入而更高）

**基于 MRR 的路由：**

| 账户 MRR | 取消流程 |
|----------|---------|
| <¥100/月 | 自动化流程 + 挽留方案 |
| ¥100-500/月 | 自动化 + 标记客户成功团队跟进 |
| ¥500-2,000/月 | 取消前转交客户成功团队 |
| ¥2,000+/月 | 禁止自助取消，要求联系客户成功团队 |

---

### 免费增值模式 / 免费转付费

用户取消付费、回到免费版。心理动因不同——他们不是离开，而是降级。

**流程结构：**
```
取消按钮 → "切换到免费版？"提示 → 退出调查（如仍要取消） → 方案 → 确认
```

**特点：**
- 首先展示免费版作为选项（而不是挽留方案）
- 展示免费版保留什么 vs. 失去什么
- "挽留成功"是留在免费版，而不是完全流失
- 追踪免费版用户，为未来的升级活动做准备

## 按计费周期分类

### 月付用户

- 对价格更敏感，承诺期更短
- 折扣方案效果好（2-3 个月 7-8 折）
- 暂停方案有效（1-2 个月）
- 建议以折扣价转年付

**方案优先级：**
1. 折扣（原因为价格）
2. 暂停（原因为用得少 / 临时需求）
3. 转年付（活跃但价格敏感）

### 年付用户

- 承诺度更高，取消通常有更强的理由
- 退款预期很重要
- 挽留窗口更长（他们已经付过款）
- 人工跟进更值得（更高的 LTV）

**方案优先级：**
1. 暂停剩余期限（临时需求）
2. 调整套餐 + 下次续费的抵用额度
3. 客户成功团队人工跟进
4. 部分退款 + 降级（好于全额退款 + 取消）

**退款处理：**
- 如果剩余时间较长，提供按比例退款
- 如果剩余不到 3 个月，"暂停至续费日"
- 大方一些——糟糕的退款体验会制造传播负面口碑的用户

## 挽留方案模式

### 阶梯式折扣

不要一上来就给最大折扣。逐步升级：

```
点击取消 → 85 折 → 仍要取消 → 75 折 → 仍要取消 → 放行
```

**规则：**
- 每次取消会话最多 2 次折扣方案
- 永远不超过 7 折（更大的折扣会培养"取消换折扣"的习惯）
- 折扣有时限（2-3 个月，之后恢复原价）
- 追踪接受折扣的用户——如果他们在恢复原价后再次取消，不再提供折扣

### 暂停方案手册

暂停通常比折扣更好，因为它不会贬低你的产品价值。

**实施方案：**

| 设置项 | 建议 |
|--------|------|
| 暂停时长选项 | 1 个月、2 个月、3 个月 |
| 默认选择 | 1 个月（最短） |
| 最长暂停 | 3 个月（更长的暂停很少回来） |
| 暂停期间 | 保留数据，移除访问权限 |
| 恢复方式 | 自动恢复，提前 7 天发邮件通知 |
| 重复暂停 | 每 12 个月允许暂停 1 次 |

**暂停恢复序列：**
- 恢复前第 7 天："你的暂停将在 7 天后结束。我们一直在更新——看看有什么新变化。"
- 恢复前第 1 天："明天就回来了！看看有什么在等你。"
- 恢复当天："欢迎回来！这是新功能快速导览。"

### 降级路径

对于多套餐产品，降级是最强的挽留手段：

```
┌─────────────────────────────────────────┐
│  在你离开之前，考虑调整一下你的套餐？     │
│                                         │
│  当前：专业版（¥49/月）                  │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ 切换到入门版（¥19/月）           │    │
│  │                                 │    │
│  │ ✓ 保留：项目、集成              │    │
│  │ ✗ 失去：高级分析、              │    │
│  │         团队功能                │    │
│  │                                 │    │
│  │ [切换到入门版]                   │    │
│  └─────────────────────────────────┘    │
│                                         │
│  [不了，继续取消]                        │
└─────────────────────────────────────────┘
```

**降级最佳实践：**
- 清晰展示保留什么和失去什么
- 用 ✓ 和 ✗ 标记便于快速浏览
- 在低套餐上也保留用户数据
- 降级后至少 30 天内不要推送升级提示

### 竞品转换应对

当取消原因是"转向其他工具"时：

1. **询问是哪个竞品**（可选，不强制）
2. **展示对比**（如果有准备好的功能对比表）
3. **提供迁移优惠**（"我们匹配他们的价格 3 个月"）
4. **邀请反馈电话**（"15 分钟聊聊我们还缺什么"）

这些数据对产品和市场团队来说非常珍贵。

## 取消后体验

取消后的体验影响：
- 召回潜力
- 口碑传播
- 评价倾向

### 确认页面

```
你的订阅已取消。

接下来会发生：
• 你的访问权限持续到 [计费周期结束日]
• 你的数据将保留 90 天
• 你可以随时在账户设置中重新激活

[重新激活我的账户]

我们期待你的回归。我们会持续根据像你这样的用户反馈来改进。
```

### 取消后序列

| 时间 | 操作 |
|------|------|
| 立即 | 发送确认邮件，包含访问截止日期 |
| 第 1 天 | （什么都不做——不要表现得很急切） |
| 第 7 天 | NPS/满意度调查，了解整体体验 |
| 第 30 天 | "有什么新的"邮件，展示近期改进 |
| 第 60 天 | 如果解决了他们取消的具体原因，定向通知 |
| 第 90 天 | 最终召回，附带特别优惠 |

**召回邮件序列详情：** 设计 3 封取消后召回邮件（第 7 天、第 30 天、第 60 天）。

## 分群规则

最有效的取消流程使用分群策略，为不同用户展示不同方案。

### 分群维度

| 维度 | 重要性 |
|------|--------|
| 套餐 / MRR | 高价值用户获得人工跟进 |
| 用户龄 | 老用户获得更优厚的方案 |
| 使用频率 | 高频用户和沉默用户需要不同的话术 |
| 计费周期 | 月付 vs. 年付需要不同策略 |
| 历史挽留记录 | 不要给重复取消的用户再次提供相同折扣 |
| 取消原因 | 决定展示哪个方案（核心映射） |

### 按分群定制流程

**新用户（< 30 天）：**
- 他们还没激活。挽留靠引导，不靠折扣。
- 方案：免费引导培训、设置帮助、延长试用
- 询问："你原本想用它做什么？"（了解缺什么）

**活跃用户因价格取消：**
- 他们喜欢产品但觉得不值。
- 方案：折扣、转年付、降级
- 挽留潜力高

**沉默用户（30 天以上未登录）：**
- 他们忘了你。折扣拉不回来。
- 方案：暂停订阅、"发生了什么变化？"对话
- 挽留潜力低——重点是了解原因

**深度用户转向竞品：**
- 他们在主动选择其他方案。
- 方案：竞品价格匹配、反馈电话、产品路线图预览
- 挽留潜力中等——取决于原因

## 实施清单

### 第一阶段：基础搭建（第 1 周）
- [ ] 添加取消流程（调查 + 1 个方案 + 确认）
- [ ] 搭建退出调查，包含 5-7 个原因分类
- [ ] 为每个原因映射一个方案（简单的 1:1 映射）
- [ ] 在分析工具中追踪取消原因和挽留率
- [ ] 启用银行卡到期前的预防性邮件

### 第二阶段：优化（第 2-4 周）
- [ ] 添加备选方案（每个原因一个主方案 + 一个备选）
- [ ] 实现暂停订阅功能
- [ ] 搭建催款邮件序列（10 天内 4 封邮件）
- [ ] 启用智能重试（Stripe Smart Retries 或同类功能）
- [ ] 为高价值账户添加基于 MRR 的路由

### 第三阶段：进阶（第 2 个月+）
- [ ] 基于使用信号构建健康度评分
- [ ] 搭建主动干预触发机制
- [ ] A/B 测试折扣金额和方案类型
- [ ] 按套餐、用户龄和使用频率进行流程分群
- [ ] 取消后召回序列（设计多触点邮件系列）
- [ ] 队列分析：按渠道、套餐、用户龄分析流失

## 合规说明

### FTC 一键取消规则（美国）
- 取消必须和注册一样简单
- 如果注册是在线完成的，不能要求电话取消
- 不能添加过多步骤来阻止取消
- 挽留方案是允许的，但"继续取消"选项必须清晰可见

### GDPR / 数据保留（欧盟）
- 告知用户取消后的数据保留期限
- 在账户删除前提供数据导出选项
- 在 30 天内响应删除请求
- 未经同意不得将取消后的数据用于营销

### 通用最佳实践
- 始终展示清晰的取消完成路径
- 绝不隐藏取消按钮（暗黑模式）
- 即使挽留流程出错，也要完成取消处理
- 通过邮件确认取消
