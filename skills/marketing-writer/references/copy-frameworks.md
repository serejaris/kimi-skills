## 落地页板块类型

### 核心板块

**首屏（首屏区域）**
- 标题 + 副标题
- 主要CTA
- 配图（产品截图、主视觉图）
- 可选：社会认同条

**社会认同条**
- 客户Logo（知名度 > 数量）
- 关键数据（"10,000+ 团队在用"）
- 星级评分及评价数量
- 简短用户评价摘要

**痛点/问题板块**
- 比用户自己更精准地描述他们的问题
- 引发共鸣（"这完全就是我的情况"）
- 暗示不解决这个问题的代价

**解决方案/利益板块**
- 从问题过渡到你的解决方案
- 3-5个核心利益点（不是10个）
- 每个利益点：标题 + 说明 + 佐证（如有）

**使用流程**
- 3-4个编号步骤
- 降低感知复杂度
- 每步：动作 + 结果

**尾部CTA板块**
- 重申价值主张
- 重复主要CTA
- 消除顾虑（保障承诺、免费试用）

---

### 辅助板块

**用户评价**
- 完整引用，附姓名、职位、公司
- 尽量附照片
- 具体成果优于笼统称赞
- 形式：引用卡片、视频、社交媒体嵌入

**案例研究**
- 问题 → 解决方案 → 成果
- 具体的数据和结果
- 客户名称和背景
- 可以是摘要加"查看详情"链接

**使用场景**
- 产品的不同使用方式
- 帮助访客自我对号入座
- "适合需要X的营销人员"格式

**目标人群/"为谁打造"板块**
- 明确标出目标受众
- "完美适合[角色]"模块
- 解答"这适合我吗？"的疑问

**常见问题板块**
- 解答常见异议
- 有利于SEO
- 减轻客服压力
- 5-10个最常见问题

**竞品对比板块**
- 与竞品对比（可以点名也可以不点名）
- 与现状对比（电子表格、手动流程）
- 表格或并列对比形式

**集成/合作伙伴**
- 展示对接工具的Logo
- "与你的技术栈无缝协作"
- 增强可信度

**创始人故事/品牌宣言**
- 你为什么做这件事
- 你的信念是什么
- 建立情感连接
- 与没有温度的竞品形成差异

**演示/产品导览**
- 交互式演示
- 视频讲解
- GIF预览
- 展示产品实际操作

**定价预览**
- 即使不是定价页也可以放
- 起步价或"低至 ¥X/月"
- 推动决策者行动

**保障/消除顾虑**
- 无条件退款保证
- 免费试用条款
- "随时取消"
- 降低决策阻力

**数据亮点板块**
- 增强可信度的关键数据
- "10,000+ 客户"
- "4.9/5 评分"
- "为客户节省 $200万"

## 页面结构模板

### 功能堆叠型页面（较弱）

```
1. 首屏
2. 功能1
3. 功能2
4. 功能3
5. 功能4
6. CTA
```

这只是一个列表，不是有说服力的叙事。

---

### 叙事驱动型页面（推荐）

```
1. 首屏——清晰的价值主张
2. 社会认同条（Logo或数据）
3. 痛点/问题板块
4. 使用流程（3步）
5. 核心利益点（2-3个，不是10个）
6. 用户评价
7. 使用场景或目标人群
8. 竞品对比
9. 案例研究摘要
10. 常见问题
11. 尾部CTA + 保障承诺
```

这是在讲故事，同时化解异议。

---

### 精简落地页

```
1. 首屏（标题、副标题、CTA、配图）
2. 社会认同条
3. 3个核心利益点 + 图标
4. 用户评价
5. 使用流程（3步）
6. 尾部CTA + 保障承诺
```

适合广告落地页等需要简洁的场景。

---

### 企业/B2B落地页

```
1. 首屏（成果导向标题）
2. Logo条（知名企业）
3. 痛点板块（业务层面）
4. 解决方案概述
5. 按角色/部门划分的使用场景
6. 安全/合规板块
7. 集成工具Logo
8. 带数据的案例研究
9. ROI/价值板块
10. 联系/预约演示CTA
```

针对企业级买家的关注点。

---

### 产品发布页

```
1. 首屏——发布公告
2. 视频演示或功能讲解
3. 功能亮点（3-5个）
4. 前后对比
5. 早期用户评价
6. 发布定价或抢先体验优惠
7. 带紧迫感的CTA
```

适合产品发布、上线公告等场景。

## 板块写作技巧

### 痛点板块

可以用这些短语开头：
- "你一定有过这种感觉……"
- "如果你和大多数[角色]一样……"
- "每天，[受众]都在为……发愁"
- "我们都经历过……"

然后描述：
- 具体的挫败感
- 浪费的时间/金钱
- 对工作/生活的影响

### 利益板块

每个利益点应包含：
- **标题**：用户得到的结果
- **正文**：如何实现（1-2句话）
- **佐证**：数据、用户评价或实例（可选）

### 使用流程板块

每个步骤应该：
- **编号**：营造推进感
- **简洁动词**："连接""设置""获取"
- **结果导向**：这一步能得到什么

示例：
1. 连接你的工具（只需2分钟）
2. 设置你的偏好
3. 每周一自动收到报告

### 用户评价筛选

好的用户评价应包含：
- 具体成果（"转化率提升了32%"）
- 前后对比（"我们以前要花好几个小时……"）
- 姓名 + 公司，增加可信度
- 有值得引用的具体内容

避免只说以下内容的评价：
- "好产品！"
- "喜欢！"
- "很好用！"
