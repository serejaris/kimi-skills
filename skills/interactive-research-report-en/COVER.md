# COVER.md · The four-state cover engine and the "theme atom" abstraction

The cover is not decoration — it is the **thesis map**. Reference `references/gallery/cover_{a,b,c}_*.png` and `cover_d_unbox.png`.

## 1 · The theme atom (first step of re-theming)

One report = **one physical atom**: the most decomposable, real-material, instantly recognizable entity in the industry. The four cover variants are all different views of the **same physical atom**, not four different things:

- **A infinite recursion** = self-similar zoom of the physical atom
- **B realistic explosion** = layered disassembly of the physical atom
- **C blueprint wireframe** = engineering-drawing rendering of B
- **D unboxing opener** = B's explosion + unboxing animation (the unrestrained impact version)

| Theme | Physical atom | Exploded layers = thesis-pillar examples |
|---|---|---|
| AI chips | accelerator package / die | cold plate = power / HBM = tightest gate / die = same-generation process / interposer = CoWoS / substrate = interconnect |
| Hog cycle | industrial farm unit | roof = policy / ventilation layer = efficiency / barns = inventory / feed lines = feed cost / foundation = land & environment |
| Power/energy | substation or generating unit | cooling tower = water / turbine = fuel / generator = equipment cycle / switchyard = grid connection |
| Shipping/logistics | container ship in layers | deck boxes = freight rates / holds = capacity / fuel tanks = oil price / hull = shipbuilding cycle |
| Cloud/SaaS | data-center rack | power supply = cost / servers = capex / network = interconnect / facility = siting |

Once the physical atom is chosen, all four states follow automatically. A/B/C are generated by the geometry engine in §2; D reuses B's explosion engine plus an unboxing timeline.

## 2 · Geometry engine (shared by B/C/D; A is independent)

```
Projection  pt(x,y,z): rx=x·cosYaw−y·sinYaw; ry=x·sinYaw+y·cosYaw
       → { cx+rx·u, cy+ry·u·0.5 − z·u }          // axonometric, z lifts
Turntable yaw = π/4 ± 0.3·sin(t·0.11) + mouse-x offset ±0.11
Layout    dual constraints: leftBound = 0.585·W (clear the left text column and chips row! 0.545 collides with buttons)
       right = W − 332 (label column); u = min((right−leftBound)/44, H·0.0132)
       u < 5.6 → drop labels, use right=W−40; cx=(leftBound+right)/2, cy=0.56H
Explosion master k(0→1, easeOutBack); per-layer lay(i)=clamp(k·1.55−i·0.17,0,1) staggered bounce
       + per-layer sin breathing float; click cover blank area to assemble/re-explode (excluding button/a/chips)
Labels    per-layer candidate anchor array, pick "rightmost projected x" → leader line to the 312px label column on the right,
       rows pushed down when spacing <34px; head/sub lines measure-truncated to column width−26
Copy      bottom-right caption / signature-strip text must switch with state (EXPLODED↔ASSEMBLED↔UNBOXING)

       re-fit() when activated (measures 0×0 during display:none)

       under reduced motion, defer the one-shot frame with requestAnimationFrame until after layout reflow, then fit+draw

```

## 3 · Variant A · Infinite recursion (**self-similarity of the object, not a loop of logic**)

**The one most often done wrong: what recurses is "the object itself", not "some causal/mechanism logic".**

The chip reference implementation draws a **recognizable die** directly: the camera keeps zooming out, the current die shrinks into the center cell of a larger die, a new ring of cells materializes in BFS birth order from the center outward, a seamless 12s/layer loop. What it recurses is **the physical chip** — not the logic chain "demand → expansion → supply → price". Don't make A into an animated flowchart/causal loop; that's not infinite recursion, that's a mechanism diagram with a zoom slapped on.

**The right way**:
1. Take the physical atom's **plan-view mask** (chip: die body + edge pins; farm: barn rectangles + two wings of feed-tower rows; port: berth grid + quay cranes). The mask must be instantly recognizable as the object, not a semantic-less grid.
2. Pre-render 8–12 **interior-texture tiles** of the object (circuit traces / barn floor plans: aisles + rails + hog dot-matrix + doorway gaps / yard container cells); tiles must be the object's **real interior** — change industry, change texture.
3. BFS birth order from the center outward + electric-blue flash at birth; camera zooms out, zoom ratio R=grid/0.78; a blue four-corner viewfinder frames the current atom "shrinking into a cell"; the center cell stays empty for the next recursion layer.
4. The recursion's **meaning** comes from "what the larger array represents" — a larger capacity map / a fab's wafer / a port's container yard — and the caption spells it out ("every farm is one cell on a larger capacity map — scale, all the way up").

**Four rejection tests**:
1. Remove all text and look at the first frame — can you recognize the industry's **object**? (No = mask/tiles too abstract, redraw)
2. Is the recursion unit a **recognizable entity** (die / farm / container), not a generic grid?
3. Swap in another industry's copy — does the graphic still hold? If yes = over-templated, textures not industry-specific enough.
4. Is the zoom truly self-similar and seamless? (The current atom must shrink precisely into one cell of the layer above; the seam must not jump)

**Lessons**: no wavefront shockwave rings — flashing arcs look cheap, the BFS birth order already is the wave; the viewfinder's opacity only fades with phase, never flickers with time.

## 4 · Variant B · Realistic-material exploded view

Each layer = axonometric box (painter-sorted sides + top) + material recipe. Generic material functions:
- **Side volumetric gradient**: top edge white .30 → mid transparent → bottom edge ink .10 (parameterized sideGloss/sideShade)
- **Top-face angled lighting** glossTop + **top-edge bright bevel** (white .55–.9)
- **sheen highlight sweep**: moving white band clipped inside the top face (metal/glass feel)
- **plateShadow inter-layer soft ellipse**: alpha=clamp(.20−sep·.007) (levitation feel)
- **ball metal spheres**: radial gradient + highlight dot (universal for solder balls/rivets/bolts)
- **Pre-rendered textures**: brushed metal (340 horizontal strokes) and speckle (±pixel dots) on offscreen canvas → createPattern (seeded RNG for reproducibility)
- **Ground shadow**: overall elliptical radial gradient below
- **identity details**: each layer gets one stroke that identifies the industry (barn pitched roof + red-brick gable + ridge ventilator, unit cooling tower, ship hatch covers) — don't let layers read as "generic boxes".

Materials use **true colors** (color discipline exempted here): chips = PCB green/gold/silicon gray/aluminum; farm = red brick/galvanized steel/concrete; ship = hull red/deck gray/container colors (desaturated). Each layer's leader-line annotation = a report thesis (HBM·BINDS HARDEST / feed lines·deepest squeeze in semantic red — the only red in the whole figure, reserved for the hardest constraint).

## 5 · Variant C · Blueprint wireframe

A drafting rendering of the same geometry engine. Rules:
- **X-ray wireframe**: no hidden-line removal. Farthest corners faint (.22), near edges solid (.7), top-face outline heaviest (.85); top face covered with a white veil (.55–.8) for front-back layering
- **Per-layer coloring**: each layer's line color = that layer's thesis color (same source as B's labels) — wireframe ≠ monochrome; layer semantics must be readable in the wireframe
- Drafting lettering: hollow circles at leader endpoints, cross grid base (52px), four-corner registration marks (electric blue), bottom-right signature strip (FIG. X · columns · SCALE NTS · CLICK TO ASSEMBLE/EXPLODE switching with state)
- Detail linework: pad hollow rings, window dashed lines, grille fine lines, all ≤1px hairline

## 6 · Variant D · Unboxing opener (**the unrestrained impact version**) `cover_d_unbox.png`

Use D when the report wants a more impactful opening. It's not new geometry — it's **B's explosion engine + an unboxing timeline**: the shipping crate sits still → the lid bursts off → five physical layers eject from the crate → settle into B's exploded state. Selecting D auto-plays it (the opener); double-click cover blank area to replay.

**Timeline** (`INTRO.t` seconds, one-shot choreography, each event fires once):
```
0.00–0.50  rattle: crate closed, small shaking builds (shakeAmp = 1.2 + 1.2·|sin(t·34)|), k=0 hidden inside
0.50       LID BURST: lid flies off — shakeAmp spikes to 15, top shock ring + wood dust (spraying up)
0.50–1.35  lid drifts up-left in screen space (-150,-40)·eoc + tumbles (-0.85rad) + fades; diagonal sealing nail strips fly with the lid
0.55–1.00  four walls alpha 1→0 fade (revealing contents); wall fall triggers a second ring + dust (outward)
0.60–2.00  k: 0→1 ejection, backOut overshoot amp=3.1 (harder than the usual 1.70); layers staggered
0.55–2.10  camera pulls in → settles: uK 1.34→1.0; yawExtra 0.55→0 (opening yaw returns to center)
1.90–2.80  labels fade in (reuse B's la=clamp((k−0.45)·2.4), appears naturally at k=1); particles settle
2.95       finishIntro: k=1, uK=1, clear particles → exactly B's exploded state
```

**Engine notes** (implemented inside cover-exploded.js; D and B share canvas and engine):
- **Screen shake**: at draw() start `ctx.save(); ctx.translate(sin(t·96)·shk, cos(t·81)·shk·0.7)`, `ctx.restore()` before wash/caption (shakes the 3D content, not the text layer); shakeAmp decays each frame `*= exp(−dt·5.5)`.
- **Camera dolly**: at camera() end `u = max(5,uu)·uK`; in the loop `yaw = π/4 + 0.14·sin(t·0.6) + yawExtra`.
- **Crate**: four painter-sorted wood-grain axonometric walls (plank seams + vertical corner irons + front-wall **upright stenciled marks** — don't rotate the stencil with the face's skew, it flips; draw upright mono directly at the near wall's bbox center: "FRAGILE / INVENTORY / THIS SIDE UP ↑"). The lid is an independent piece: z lift + screen-space translate/rotate tumble + fade, drawn last (flies on top).
- **Particles**: dust (gravity-parabola wood chips, warm wood + steel gray) + ring (elliptical shock ring, scale(1,0.5) flattened into ground perspective); both drawn inside the shaken coordinate system, after the lid.
- **Trigger API**: `COVER_X.setActive(true, {intro:true})` plays on entry; `COVER_X.playIntro()` replays; double-click cover blank `startIntro()`; click assemble/disassemble disabled during intro (`if (INTRO.on) return`).
- **reduced-motion**: D skips the whole timeline and shows B's static exploded completed frame directly (`finishIntro()` + deferred-frame draw).

**Pitfalls**: the alpha fades of lid/walls and the k ejection must overlap correctly — the lid must fly away before the roof layer reaches the crate mouth, or the lid covers the top layer passing through; the four walls stay opaque until k releases, hiding the contents (at k=0 the contents sit compactly inside the crate).

## 7 · Porting checklist

Port from the reference implementation: `cover.js` (A object-recursion engine: swap mask shape + interior-texture tiles), `cover-exploded.js` (B/D material-explosion engine + unboxing timeline: swap layer-definition array/material palette/labels/identity details), `cover-wire.js` (C wireframe engine, same geometry as B + **the four-state switcher**: D shares COVER_X with B, `setMode('d')→COVER_X.setActive(true,{intro:true})`, `useX = (m==='x'||m==='d')`). When changing A, swap only mask + tile textures, never the engine; **never turn A into a mechanism/causal loop — what recurses is always the object**.
**Three HTML/CSS things**: all four canvases (three for A/B/C + D reusing B's canvas) must be inside the `#cover-canvas*` `position:absolute; inset:0; width/height:100%` selector (miss one and it's 0×0 all blank); pill buttons data-mode (including `data-mode="d"`); switching logic lives in the last-loaded cover-wire.js.
