# Legal Doc Generator — 法律文书生成器

通过交互式问答收集关键变量，生成结构完整的法律文书初稿。支持 NDA（保密协议）、服务协议、隐私政策、合作框架协议等常见文书类型。

> **免责声明**：本工具生成的文书仅供参考，不构成法律意见。正式签署前，请务必由专业律师审核。不同司法管辖区的法律要求可能不同，请确保文书符合当地法律法规。

---

## Quick Start

用户只需说明：
1. **文书类型**：NDA / 服务协议 / 隐私政策 / 合作框架协议
2. **基本场景**：签约双方、合作目的、适用范围

示例：
> "帮我生成一份双向 NDA，我方是一家科技公司，对方是潜在投资人，主要保护技术方案和商业数据。"

## 二、交互式问答流程（Variable Collection SOP）

### 2.1 通用变量（所有文书类型必收集）

在生成任何法律文书前，必须通过问答确认以下变量：

| 序号 | 变量 | 问题示例 | 说明 |
|------|------|---------|------|
| 1 | 甲方名称 | "甲方（披露方/服务提供方）的全称是什么？" | 公司全称或自然人姓名 |
| 2 | 乙方名称 | "乙方（接收方/客户方）的全称是什么？" | 同上 |
| 3 | 签署日期 | "协议签署日期是哪天？（留空则填写签署当日）" | 格式：YYYY年MM月DD日 |
| 4 | 适用法律 | "本协议适用哪个司法管辖区的法律？（如：中华人民共和国法律）" | 默认中华人民共和国法律 |
| 5 | 争议解决 | "争议解决方式选择仲裁还是诉讼？如选仲裁，指定哪个仲裁机构？" | 仲裁 / 诉讼 |
| 6 | 语言 | "文书使用中文还是英文？" | 中文 / 英文 / 中英双语 |

### 2.2 NDA（保密协议）专属变量

| 序号 | 变量 | 问题示例 | 说明 |
|------|------|---------|------|
| N1 | 保密方向 | "单向保密（一方披露）还是双向保密（双方互相披露）？" | 单向 / 双向 |
| N2 | 保密信息范围 | "需要保护哪些类型的信息？（如：技术方案、商业数据、客户列表、财务信息）" | 可多选 |
| N3 | 保密期限 | "保密义务持续多长时间？（如：协议终止后2年）" | 通常 1-5 年 |
| N4 | 排除信息 | "是否有不属于保密信息的例外情况？（如：公开信息、自行研发的信息）" | 标准排除条款 |
| N5 | 违约责任 | "违约金额或赔偿方式有无特殊约定？" | 具体金额 / 实际损失赔偿 |

### 2.3 服务协议专属变量

| 序号 | 变量 | 问题示例 | 说明 |
|------|------|---------|------|
| S1 | 服务内容 | "服务的具体内容是什么？请简要描述。" | 详细服务范围 |
| S2 | 服务期限 | "服务期限是多长？起止日期？" | 固定期限 / 按项目 |
| S3 | 服务费用 | "服务费用是多少？支付方式？（如：月付、一次性、里程碑付款）" | 金额 + 支付方式 |
| S4 | 交付标准 | "交付物或验收标准是什么？" | 可量化的标准 |
| S5 | 知识产权 | "服务过程中产生的知识产权归属谁？" | 甲方 / 乙方 / 共有 |
| S6 | 终止条件 | "任一方可以提前终止吗？需要多少天提前通知？" | 提前通知天数 |

### 2.4 隐私政策专属变量

| 序号 | 变量 | 问题示例 | 说明 |
|------|------|---------|------|
| P1 | 产品名称 | "这份隐私政策适用于哪个产品/网站/App？" | 产品全称 + 网址 |
| P2 | 收集数据类型 | "收集哪些用户数据？（如：姓名、邮箱、手机号、位置信息、浏览记录）" | 列举所有类型 |
| P3 | 数据用途 | "收集这些数据的目的是什么？（如：提供服务、个性化推荐、数据分析）" | 逐一对应 |
| P4 | 第三方共享 | "是否会与第三方共享用户数据？如是，哪些第三方？" | 广告平台 / 分析工具 / 不共享 |
| P5 | 数据存储 | "数据存储在哪里？保留多长时间？" | 服务器位置 + 保留期限 |
| P6 | 用户权利 | "用户是否可以查看、修改、删除自己的数据？" | 通常都需要支持 |
| P7 | 合规要求 | "需要符合哪些法规？（如：GDPR、中国个人信息保护法、CCPA）" | 可多选 |

### 2.5 合作框架协议专属变量

| 序号 | 变量 | 问题示例 | 说明 |
|------|------|---------|------|
| C1 | 合作目的 | "双方合作的目标和范围是什么？" | 简明描述 |
| C2 | 各方职责 | "甲方和乙方各自的主要职责是什么？" | 分别列出 |
| C3 | 合作期限 | "合作期限多长？是否自动续期？" | 固定期限 / 自动续期 |
| C4 | 收益分配 | "合作收益如何分配？" | 比例 / 固定金额 |
| C5 | 排他性 | "是否要求排他合作？（即合作期间不得与第三方开展同类合作）" | 排他 / 非排他 |

## 三、文书模板库

### 3.1 NDA（保密协议）模板

**中文模板**：

```
                        保密协议

协议编号：[编号]
签署日期：[签署日期]

甲方（[披露方/甲方角色]）：[甲方名称]
注册地址：[甲方地址]
法定代表人：[甲方法定代表人]

乙方（[接收方/乙方角色]）：[乙方名称]
注册地址：[乙方地址]
法定代表人：[乙方法定代表人]

鉴于甲乙双方拟就[合作事项]进行商务洽谈与合作探讨，在此过程中
可能涉及双方各自的保密信息，为保护双方合法权益，经友好协商，
达成如下协议：

第一条 保密信息的定义

1.1 本协议所称"保密信息"是指一方（"披露方"）向另一方（"接收方"）
披露的、与[保密信息范围]相关的所有非公开信息，包括但不限于：
  （一）技术信息：技术方案、算法、源代码、设计文档、技术诀窍等；
  （二）商业信息：商业计划、客户列表、定价策略、财务数据等；
  （三）其他信息：双方协商确定的其他应予保密的信息。

1.2 保密信息的形式包括但不限于：书面文件、电子文档、口头陈述、
演示材料、样品及实物。

第二条 保密信息的排除

2.1 下列信息不属于本协议所称的保密信息：
  （一）在披露时已属于公知信息的；
  （二）在披露后非因接收方过错而成为公知信息的；
  （三）接收方在接收前已合法拥有且无保密义务的；
  （四）接收方从有权披露的第三方处合法获得的；
  （五）接收方独立开发且不依赖保密信息的。

第三条 保密义务

3.1 接收方应对保密信息承担保密义务，未经披露方事先书面同意，
不得向任何第三方披露、公开或泄露保密信息。

3.2 接收方仅可在与[合作事项]相关的目的范围内使用保密信息，
不得用于任何其他目的。

3.3 接收方应采取合理的安全措施保护保密信息，其保护程度不低于
接收方保护自身同类保密信息的程度。

3.4 接收方可在必要范围内向其员工、顾问、关联方披露保密信息，
但应确保上述人员知悉并遵守本协议项下的保密义务。

第四条 保密期限

4.1 本协议项下的保密义务自本协议签署之日起生效，持续至
[保密期限]届满之日止。

4.2 协议终止或期满后，接收方应在[30]日内返还或销毁所有保密信息
的载体（包括副本），并书面确认已完成销毁。

第五条 违约责任

5.1 任何一方违反本协议的保密义务，应赔偿对方因此遭受的全部
直接损失。[违约责任条款]

5.2 守约方有权要求违约方立即停止违约行为，并采取合理措施消除
影响。

第六条 一般条款

6.1 本协议适用[适用法律]。

6.2 因本协议引起的或与本协议有关的任何争议，[争议解决条款]。

6.3 本协议一式两份，甲乙双方各执一份，具有同等法律效力。

6.4 本协议的修改须经双方书面同意。


甲方（盖章）：                    乙方（盖章）：

授权代表签字：                    授权代表签字：

日期：                            日期：
```

### 3.3 隐私政策模板

```

最后更新日期：[日期]

[产品名称]（以下简称"我们"）非常重视用户的隐私保护。本隐私政策

一、我们收集的信息

1. 您主动提供的信息
[根据 P2 变量列举，例如：]
  - 账户注册信息：姓名、邮箱地址、手机号码
  - 身份验证信息：身份证号码（如适用）
  - 支付信息：银行卡号、支付账户（如适用）

2. 我们自动收集的信息
  - 设备信息：设备型号、操作系统版本、唯一设备标识符
  - 日志信息：IP 地址、访问时间、浏览页面、操作记录
  - 位置信息：[如适用，说明是否收集精确或模糊位置]

3. 来自第三方的信息
  - [如适用，说明从哪些第三方获取何种信息]

二、我们如何使用您的信息

我们收集您的信息用于以下目的：
[根据 P3 变量列举，例如：]
  - 提供、维护和改进我们的服务
  - 处理您的交易和发送相关通知
  - 向您发送服务更新和营销信息（经您同意）
  - 进行数据分析以改善用户体验
  - 预防欺诈和保障安全

三、信息的共享与披露

[根据 P4 变量填写]
我们不会出售您的个人信息。在以下情况下，我们可能共享您的信息：
  - 经您明确同意的；
  - 与我们的服务提供商共享，以便其协助我们运营（如云存储、

  - 法律法规要求或政府部门依法要求的；
  - 为保护我们、用户或公众的权利、财产或安全所必需的。

四、信息的存储与保护

4.1 存储地点：您的个人信息存储在[数据存储位置]的服务器上。

4.2 存储期限：我们将在[数据保留期限]内保留您的个人信息，
或在实现本隐私政策所述目的所需的期限内保留。

4.3 安全措施：我们采取合理的技术和管理措施保护您的个人信息，
包括但不限于数据加密、访问控制和安全审计。

五、您的权利

根据适用法律，您享有以下权利：
  - 访问权：查看我们持有的您的个人信息
  - 更正权：更正不准确或不完整的个人信息
  - 删除权：在特定情况下要求删除您的个人信息
  - 撤回同意：撤回您此前给予的处理同意
  - 数据可携带权：在技术可行的情况下，获取您的个人信息副本

如需行使上述权利，请通过[联系方式]与我们联系。

六、未成年人保护

我们的服务面向[年龄要求，如：16岁以上]用户。我们不会故意收集
未满[年龄]周岁未成年人的个人信息。

七、隐私政策的更新

我们可能会不时更新本隐私政策。更新后的政策将在[产品名称]上发布，
重大变更时我们将通过[通知方式]通知您。

八、联系我们

如您对本隐私政策有任何疑问，请通过以下方式联系我们：
  - 电子邮件：[邮箱地址]
  - 联系电话：[电话号码]
  - 通信地址：[公司地址]
```

## 四、条款要点解读（帮助用户理解每个关键条款）

| 条款 | 作用 | 缺失的风险 |
|------|------|-----------|
| 保密信息定义 | 明确保护范围，避免争议 | 保护范围模糊，难以追责 |
| 排除条款 | 排除公开信息等，避免义务过宽 | 接收方承担不合理的保密负担 |
| 保密期限 | 明确义务存续时间 | 可能被认定为永久保密或无效 |
| 违约责任 | 明确违约后的法律后果 | 违约后缺乏救济手段 |
| 争议解决 | 预设纠纷解决路径 | 发生纠纷时增加诉讼成本 |
| 知识产权归属 | 明确成果物归属方 | 产生 IP 归属纠纷 |
| 不可抗力 | 界定免责情形 | 天灾人祸时责任不清 |
| 通知条款 | 明确有效通知方式 | 关键通知无法送达 |

## 六、Agent 行为指南

### 6.1 整体流程

```
用户发起请求
    ↓
① 确认文书类型（NDA / 服务协议 / 隐私政策 / 合作框架协议）
    ↓
② 收集通用变量（甲乙方、日期、适用法律、争议解决、语言）
    ↓
③ 收集专属变量（根据文书类型提问对应变量表）
    ↓
④ 生成文书初稿（套用对应模板，填入变量）
    ↓
⑤ 输出并说明（标注所有待确认的占位符，提醒律师审核）
    ↓
⑥ 根据用户反馈修改（可多轮迭代）
```

### 6.2 问答策略

- **分批提问**：不要一次列出所有问题，按"通用变量 → 专属变量"分 2-3 轮提问
- **提供默认值**：对于常见选项（如适用法律、保密期限），给出推荐默认值
- **用通俗语言**：避免法律术语堆砌，必要时附带简要解释
- **缺失信息用占位符**：用户无法提供的信息，用 `[待填写：xxx]` 标注
- **多选项引导**：对于有固定选项的变量（如单向/双向保密），直接给出选项让用户选择

### 6.3 输出规范

- 文书格式整齐，条款编号清晰
- 所有变量位置用方括号标注，便于用户查找和填写
- 中文文书使用中文标点，英文文书使用英文标点
- 文末附带免责声明，提醒用户寻求专业法律意见
- 列出所有未确认的占位符清单，方便用户逐一核实

### 6.4 常见追问处理

| 用户追问 | 应对方式 |
|---------|---------|
| "这个条款有什么用？" | 参照第四节"条款要点解读"做通俗解释 |
| "违约金一般写多少？" | 说明行业惯例范围，但强调需律师确认 |
| "能不能加个竞业限制条款？" | 可以，补充竞业限制相关条款 |
| "这个协议在国外能用吗？" | 提醒跨境协议需考虑对方国家法律，建议咨询涉外律师 |
| "帮我改成英文版" | 翻译并调整为英文法律文书惯用表达 |
