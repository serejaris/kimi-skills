# 市场定位框架

分析竞品定位、制定自身定位策略并识别差异化机会的综合框架。使用这些框架来绘制竞争格局地图、构建定位声明，并发现竞品的信息传递弱点。

## 概述

定位是指你的产品或品牌在目标受众心中相对于竞品所占据的独特位置。强有力的定位能让你的价值主张清晰、差异化显而易见，并让目标受众感到被理解。

本参考资料涵盖：
1. 定位声明模板
2. 2×2 定位地图方法论

## 1. 定位声明模板

### 经典定位声明

各行业通用的基础模板：

```
For [target audience],
[product/brand] is the [category]
that [key benefit / point of difference]
because [reason to believe / proof point].
```

**示例（SEO 工具）：**
> 对于成长期的 SaaS 营销人员，Clearscope 是一个内容优化平台，确保每篇文章都能登上搜索首页，因为它利用实时 SERP 数据和 NLP 技术将内容与排名竞争对手进行比对评分。

### 扩展定位声明

增加了竞争背景和情感共鸣：

```
For [target audience] who [situation/need],
[product/brand] is the [category]
that [functional benefit]
unlike [competitive alternative]
because [unique capability / proof].
This matters because [emotional benefit / outcome].
```

**示例：**
> 对于难以让新文章持续获得排名的内容团队，MarketMuse 是一个内容规划平台，它能精确识别需要覆盖的主题。与手动关键词研究工具不同，它使用基于排名内容训练的 AI 驱动主题模型。这意味着团队不再凭感觉写作，而是带着信心发布内容。

### 前后对比定位（Before/After/Bridge）

聚焦于转变：

```
BEFORE: [Current painful state for target audience]
AFTER:  [Desired improved state]
BRIDGE: [Your product] makes this possible by [mechanism].
```

### 痛点放大定位（Problem-Agitation-Solution）

聚焦于痛点放大：

```
PROBLEM:    [What the target audience struggles with]
AGITATION:  [Why this problem is worse than they think — consequences, hidden costs]
SOLUTION:   [How your product solves it differently than alternatives]
```

## 2. 定位地图（2×2 矩阵）方法论

### 如何构建定位地图

定位地图在两个关键维度上标注竞品位置，从而可视化竞争格局并识别空白定位空间。

#### 第一步：选择坐标轴

选择对目标受众最重要的两个维度。坐标轴应满足：
- **对买家有意义**（而非内部指标）
- **有区分度**（竞品应分布在地图各处）
- **相互独立**（两个维度不应高度相关）

#### SEO/营销领域常用坐标轴组合

| 坐标轴组合 | X 轴 | Y 轴 | 适用场景 |
|-----------|--------|--------|---------|
| 价值定位 | 价格（低 → 高） | 功能（基础 → 高级） | 了解市场分层 |
| 用户体验定位 | 易用性（复杂 → 简单） | 能力（有限 → 全面） | 评估易用性/功能权衡 |
| 受众定位 | 中小企业 ← → 大型企业 | 单点工具 ← → 全平台 | 识别细分市场空白 |
| 创新定位 | 成熟/稳定 ← → 创新/前沿 | 垂直 ← → 通用 | 把握市场时机 |
| 内容定位 | 数据驱动 ← → 观点驱动 | 入门级 ← → 专家级 | 内容策略差异化 |

#### 第二步：标注竞品

将每个竞品（包括你自己）基于客观评估标注在地图上。

```
              HIGH CAPABILITY
                    │
         Enterprise │  All-in-One
         Suites     │  Platforms
                    │
 LOW ───────────────┼─────────────── HIGH
 PRICE              │                PRICE
                    │
         Free/Basic │  Premium
         Tools      │  Specialists
                    │
              LOW CAPABILITY
```

#### 第三步：识别空白区域

寻找以下象限或区域：
- **无竞品存在**（市场机会）
- **仅有弱势竞品**（替代机会）
- **竞品扎堆**（差异化挑战——避开或重新定义）

#### 第四步：选择你的定位

选择符合以下条件的定位：
- 与你的实际产品能力相符
- 服务于未被充分满足的受众群体
- 可防御（竞品难以模仿）
- 能与清晰的价值叙事相连

### 定位地图反面教材

| 反面模式 | 描述 | 改进方法 |
|-------------|------------|-----|
| 愿望式标注 | 把自己标在想去的位置，而非实际位置 | 诚实评估当前位置；制定通往目标位置的路线图 |
| 虚荣坐标轴 | 选择让自己好看但买家不关心的维度 | 通过客户调研验证坐标轴 |
| 遗漏竞品 | 只标注直接竞品，遗漏替代方案 | 纳入间接竞品和替代性解决方案 |
| 静态绘制 | 只画一次从不更新 | 每季度刷新以反映市场变化 |

## 3. 品类策略选项

你如何定义所属品类，决定了你的竞争集合和定位灵活度。

### 策略 1：赢下现有品类

**适用场景：** 你在一个成熟品类中拥有真正优越的产品。
**做法：** 接受现有品类定义，以"做得最好"来竞争。
**信息传递：** "面向 [受众] 的最佳 [品类]"
**风险：** 与成熟玩家的正面竞争。

**示例：** "面向内容营销人员的最佳 SEO 工具"（在"SEO 工具"品类内竞争）

### 策略 2：创建新子品类

**适用场景：** 你拥有值得独立命名的真正差异化优势。
**做法：** 细分现有品类并占据新细分市场。
**信息传递：** "第一个 [新子品类]"
**风险：** 市场可能不认可该子品类。

**示例：** "内容优化平台"（"SEO 工具"中聚焦内容的子品类）

### 策略 3：创建全新品类

**适用场景：** 你的产品确实不适合现有品类。
**做法：** 定义一个新品类，并将自己定位为品类创建者。
**信息传递：** "引入 [新品类]：[定义]"
**风险：** 高额的市场教育成本；市场可能不接受该品类。

**示例：** "GEO 优化平台"（融合 SEO + AI 可见度的新品类）

### 策略 4：重新定义品类

**适用场景：** 现有品类的框架对你不利。
**做法：** 改变买家对问题的思考方式，转移评估标准。
**信息传递：** "别再纠结 [旧品类]了。开始关注 [新框架] 吧。"
**风险：** 可能让使用现有品类搜索/评估的潜在客户感到困惑。

**示例：** "不是又一个 SEO 工具——而是收入智能平台"（从 SEO 重新定义为收入）

### 品类策略决策矩阵

| 考量因素 | 赢下现有品类 | 新子品类 | 全新品类 | 重新定义 |
|--------|-------------|-----------------|-------------|---------|
| 产品差异化程度 | 低-中 | 中-高 | 非常高 | 中 |
| 市场教育成本 | 低 | 中 | 非常高 | 高 |
| 竞争烈度 | 高 | 中 | 低（初期） | 中 |
| SEO/搜索难度 | 高（竞争性词汇） | 中（新词汇） | 低（无竞争） | 中 |
| 销售周期影响 | 短（已知品类） | 中 | 长（需要教育） | 中-长 |

## 4. 差异化操作手册

### 差异化类型

并非所有差异化都同等有效。越强的差异化越难被模仿。

| 类型 | 强度 | 持久性 | 示例 |
|------|---------|-----------|---------|
| **功能差异** | 低 | 低（容易被模仿） | "我们有 X 功能" |
| **集成差异** | 中 | 中 | "我们与 Y 生态打通" |
| **体验差异** | 中-高 | 中-高 | "我们的 UX 专为 Z 工作流设计" |
| **数据/网络差异** | 高 | 高 | "来自 N 个用户的数据让结果更准" |
| **方法论差异** | 高 | 高 | "我们发明了 X 框架" |
| **使命/价值观差异** | 高 | 非常高 | "我们是唯一专注于 [使命] 的 [品类]" |

### 差异化审计模板

针对每个竞品，评估你的差异化强度：

| 维度 | 你的做法 | 竞品的做法 | 差异化强度 | 可防御性 |
|-----------|-------------|---------------------|------------------------|--------------|
| 核心技术 | [你的技术] | [对方的技术] | [弱/中/强] | [容易/难以模仿] |
| 目标受众 | [你的受众] | [对方的受众] | [弱/中/强] | [容易/难以模仿] |
| 定价模式 | [你的模式] | [对方的模式] | [弱/中/强] | [容易/难以模仿] |
| 内容/教育 | [你的方式] | [对方的方式] | [弱/中/强] | [容易/难以模仿] |
| 客户支持 | [你的方式] | [对方的方式] | [弱/中/强] | [容易/难以模仿] |
| 数据优势 | [你的数据] | [对方的数据] | [弱/中/强] | [容易/难以模仿] |
| 品牌/社区 | [你的品牌] | [对方的品牌] | [弱/中/强] | [容易/难以模仿] |

### "只有我们"测试

对每个声称的差异点，应用以下测试：

> "只有 [你的公司] [做了 X]，因为 [独特原因]。"

如果竞品也能如实做出同样的声明，那它就不是真正的差异点。修改到这句话只属于你为止。

**强示例：**
> "只有 Clearscope 能基于实时 SERP 竞品分析为内容评分，因为我们构建了基于排名内容模式训练的专有 NLP 模型。"

**弱示例：**
> "只有我们提供优质的客户支持。"（每家公司都这么说。）

### 差异化信息传递公式

```
We're the only [category] that [unique capability]
which means [customer benefit]
so you can [desired outcome].
```

## 5. 信息传递弱点分析

### 识别竞品的信息传递弱点

信息传递弱点是指竞品声称的与实际交付之间的差距，或者听起来不错但经不起推敲的信息。

#### 弱点类型

| 弱点类型 | 描述 | 如何利用 |
|-------------------|------------|---------------|
| **承诺-现实差距** | 他们声称 X 但评论说 Y | 引用评论数据；提供你的交付证明 |
| **模糊性差距** | 使用空泛表述（"业界领先"）但无证据 | 用数据支撑你的具体声明 |
| **受众错配** | 信息传递对准一个受众，但产品服务另一个 | 直接对被忽视的受众说话 |
| **过时定位** | 定位已过时；产品早已升级 | 针对他们已固化在市场认知中的旧故事进行定位 |
| **功能堆砌** | 罗列功能却不关联到成果 | 以成果为先，而非功能 |
| **价格敏感** | 回避讨论价格，暗示价格高或混乱 | 透明公开定价；制作总拥有成本对比 |

#### 竞品信息传递审计工作表

针对每个竞品，分析其在各触点的信息传递：

| 触点 | 对方的信息 | 是否有弱点？ | 你的反击信息 |
|-----------|-------------|---------------|---------------------|
| 首页主标语 | [对方的标题] | [有/无——弱在哪里？] | [你的替代角度] |
| 定价页面 | [对方如何呈现定价] | [有/无——隐藏了什么？] | [你的透明化策略] |
| 博客/内容 | [内容主题和调性] | [有/无——缺了什么？] | [你的内容差异化] |
| 社交媒体 | [社交信息传递和互动] | [有/无——什么是做样子的？] | [你的真实化策略] |
| 销售邮件/拓展 | [已知的销售话术] | [有/无——什么是强推/套路？] | [你的顾问式策略] |
| 用户评价 | [客户实际怎么说] | [有/无——与其声称是否有差距？] | [突出你的声明与评价的一致性] |

### 反定位策略

| 策略 | 适用时机 | 信息传递模式 |
|----------|-----------|-----------------|
| **对比** | 竞品占主导地位但有明显弱点 | "与 [竞品] 不同，我们 [你的优势]" |
| **侧翼** | 竞品忽略了一个有价值的细分市场 | "专为 [被忽视的细分] 而生" |
| **重构** | 竞品的优势其实是一个伪装的弱点 | "[竞品的功能] 听起来不错，但当你意识到 [负面后果]" |
| **升维** | 功能比拼注定失败 | "别比功能了。真正重要的是 [更高层面的成果]" |
| **具体化** | 竞品使用空泛表述 | "为 [具体受众] 在 [具体时间内] 实现 [具体指标]" |

### 信息传递强度测试

对每个竞品的信息传递按以下维度评分（1-5）：

| 维度 | 评估内容 | 得分 |
|-----------|-----------------|-------|
| 清晰度 | 5 秒内能否理解其价值主张？ | [1-5] |
| 具体性 | 是否使用了具体的数字和声明？ | [1-5] |
| 差异化 | 其信息是否与竞品有明显区别？ | [1-5] |
| 证据 | 声明是否有证据支撑？ | [1-5] |
| 受众契合度 | 信息是否与目标受众产生共鸣？ | [1-5] |
| 一致性 | 各触点的信息是否一致？ | [1-5] |
| 情感共鸣 | 是否触及了真实的痛点或愿景？ | [1-5] |

**总分** = 得分之和 / 35

| 分数区间 | 评估结论 |
|------------|-----------|
| 0.8-1.0 | 信息传递非常强——寻找利基角度竞争 |
| 0.6-0.79 | 信息传递扎实——利用具体弱点 |
| 0.4-0.59 | 信息传递一般——有多个攻击向量 |
| 低于 0.4 | 信息传递薄弱——有机会以清晰度取胜 |

## 综合运用这些框架

### 推荐工作流

1. **从定位地图入手**，可视化理解竞争格局
2. **分析品类策略**，决定如何定义你的竞争集合
3. **构建定位声明**，使用上述模板
4. **审计差异化**，确保你的声明可防御
5. **识别信息传递弱点**，找到攻击角度
6. **制作战斗卡片**（见 [battlecard-template.md](./battlecard-template.md)），整合所有洞察
7. **每季度回顾**，跟随竞争格局的变化
