# Speech Draft Writer

**一句话需求 → 专业演讲稿**：基于亚里士多德修辞三角框架（Ethos · Pathos · Logos），生成适配发布会、年会、TED 式等场景的演讲稿，内置停顿/语气/语速标注系统，自动预估演讲时长。

## Quick Start

用户只需说明演讲场景和主题，Agent 按 SOP 流程输出带表演标注的完整演讲稿：

```
用户：帮我写一篇年会上的CEO致辞，主题是回顾过去一年的成绩和展望未来
Agent：[按 SOP 流程输出带停顿/语气标注的演讲稿 + 预估时长]
```

## 标注系统说明

演讲稿中使用 `【】` 标注演讲提示，帮助演讲者控制节奏和情感：

| 标注类别 | 示例 | 说明 |
|----------|------|------|
| 停顿 | `【停顿 2s】` | 停顿秒数，用于制造悬念或让观众消化信息 |
| 语气 | `【语气：激昂】` `【语气：沉稳】` `【语气：温情】` | 切换情感基调 |
| 语速 | `【语速：放慢】` `【语速：加快】` `【语速：恢复正常】` | 调整语速节奏 |
| 动作 | `【扫视全场】` `【面向左侧观众】` `【走向舞台中央】` | 肢体语言与走位提示 |
| 重音 | `【重音】` | 下一句或下一个词需要加重语气 |
| 互动 | `【等待掌声】` `【举手示意观众互动】` | 与观众的互动节点 |

## 时长估算规则

- 中文演讲语速因场景而异（详见 Phase 5 的场景化参数）：发布会 230 字/分钟、年会 220、TED 式 240、仪式性 200
- 英文演讲平均语速：**130 词/分钟**（TED 风格约 140-160 词/分钟）
- 停顿时间单独累加
- 互动环节（鼓掌、提问）按标注预估额外时间
- 最终输出格式：`预估时长：XX 分 XX 秒（正文 XX 字 + 停顿 XX 秒 + 互动 XX 秒）`

## SOP 流程

### Phase 1: 场景与需求澄清

**目标**：明确演讲的场景、受众、目的和约束。

**操作步骤**：

1. **识别演讲类型**，确认属于以下哪种场景：

   | 类型 | 典型场景 | 风格特点 |
   |------|----------|----------|
   | 发布会 | 产品发布、战略发布、品牌发布 | 自信、前瞻、节奏紧凑，重 Logos + Ethos |
   | 年会/庆典 | 年会致辞、周年庆、表彰大会 | 温情、回顾、鼓舞，重 Pathos + Ethos |
   | TED 式 | 思想分享、行业洞察、个人故事 | 故事驱动、启发性强，重 Pathos + Logos |
   | 仪式性 | 开幕/闭幕词、毕业典礼、就职演讲 | 庄重、简洁、仪式感，重 Ethos |

2. **提出澄清问题**（最多 5 个关键问题）：
   - 演讲场合和活动名称是什么？
   - 演讲者的身份/职位是什么？（决定 Ethos 策略）
   - 目标听众是谁？（员工/客户/投资人/公众）
   - 演讲的核心目的是什么？（激励/告知/说服/庆祝）
   - 目标时长是多少？（如无要求，根据场景推荐）

3. **如果用户要求跳过澄清**，基于场景类型的常见默认值继续，并在稿件开头标注假设

**输出**：演讲场景摘要卡片

```
📋 演讲场景摘要
- 类型：[发布会/年会/TED式/仪式性]
- 场合：[具体活动名称]
- 演讲者：[身份/职位]
- 听众：[目标受众]
- 核心目的：[激励/告知/说服/庆祝]
- 目标时长：[X 分钟]
- 语言：[中文/英文]
```

---

### Phase 2: 修辞三角分析（Rhetorical Triangle）

**目标**：基于亚里士多德修辞三角框架，为演讲稿设计说服策略。

**修辞三角框架**：

```
         Ethos（可信度）
        /              \
       /    演讲效果     \
      /                  \
  Pathos（情感）——— Logos（逻辑）
```

**操作步骤**：

1. **Ethos 分析（可信度建立）**：
   - 演讲者的权威来源是什么？（职位、经验、成就、专业背景）
   - 如何在开场建立信任？（自我介绍策略、共同身份、谦逊开场）
   - 全篇中哪些位置需要强化可信度？（引用数据源、提及亲身经历）
   - 输出 Ethos 策略清单（3-5 条具体手段）

2. **Pathos 分析（情感共鸣）**：
   - 目标情感弧线是什么？（从哪种情感出发，经历什么变化，落在什么情感）
   - 核心情感触发点有哪些？（故事、画面、对比、悬念）
   - 听众的情感痛点/共鸣点是什么？
   - 输出情感弧线设计：`[开场情感] → [转折情感] → [高潮情感] → [结尾情感]`

3. **Logos 分析（逻辑论证）**：
   - 核心论点是什么？（一句话概括演讲的中心思想）
   - 支撑论点的论据有哪些？（数据、案例、类比、因果推理）
   - 论证结构选择：
     - 时间线结构（过去→现在→未来）
     - 问题-解决结构（痛点→方案→效果）
     - 三段论结构（三个并列论点）
   - 输出论证框架大纲

**输出**：修辞三角策略表

```
| 维度 | 核心策略 | 具体手段 | 在稿件中的位置 |
|------|----------|----------|----------------|
| Ethos | [策略] | [手段1, 手段2...] | [开场/中段/结尾] |
| Pathos | [策略] | [手段1, 手段2...] | [开场/中段/结尾] |
| Logos | [策略] | [手段1, 手段2...] | [开场/中段/结尾] |
```

---

### Phase 3: 结构设计与时间分配

**目标**：设计演讲稿的整体结构，并为每个部分分配时间。

**操作步骤**：

1. **选择演讲结构模型**（根据场景和修辞策略选择）：

   | 结构模型 | 适用场景 | 各部分占比 |
   |----------|----------|------------|
   | **经典三段式** | 通用，年会致辞 | 开场 15% · 主体 70% · 结尾 15% |
   | **英雄之旅** | TED 式，个人故事 | 日常→挑战→低谷→转折→蜕变→启示 |
   | **问题-方案-愿景** | 发布会，战略演讲 | 痛点 20% · 方案 50% · 愿景 30% |
   | **Monroe 动机序列** | 说服型演讲 | 注意→需求→满足→可视化→行动 |
   | **时间线** | 回顾总结型 | 过去 25% · 现在 35% · 未来 40% |

2. **设计段落大纲**：
   - 为每个段落标注：主题句 + 修辞重点（E/P/L）+ 预估时长
   - 标注情感弧线在结构中的对应位置
   - 标注关键停顿和互动节点的位置

3. **时间分配表**：

   ```
   | 段落 | 内容概要 | 修辞重点 | 预估字数 | 预估时长 |
   |------|----------|----------|----------|----------|
   | 开场 | [概要]   | E/P      | XXX 字   | X 分 X 秒 |
   | ...  | ...      | ...      | ...      | ...       |
   | 总计 |          |          | XXX 字   | X 分 X 秒 |
   ```

**输出**：段落大纲 + 时间分配表

---

### Phase 4: 演讲稿撰写（含标注）

**目标**：撰写完整演讲稿，嵌入停顿、语气、语速、动作标注。

**操作步骤**：

1. **开场撰写**（关键：前 30 秒决定听众注意力）：
   - 选择开场策略（根据场景）：
     - 发布会：悬念式（"今天，我们要重新定义 XXX"）
     - 年会：感恩式（"站在这里，看着你们每一张面孔..."）
     - TED 式：故事式（"三年前的一个深夜，我收到了一封邮件..."）
     - 仪式性：引用式（"正如 XXX 所说..."）
   - 开场标注要求：必须包含至少 1 个 `【停顿】` 和 1 个 `【语气】` 标注

2. **主体撰写**：
   - 按 Phase 3 的段落大纲逐段撰写
   - 每个段落之间插入过渡句 + `【停顿 2-3s】`
   - 关键数据/论点前加 `【语速：放慢】` + `【重音】`
   - 情感高潮处加 `【语气】` 切换标注
   - 故事段落使用画面感强的语言，标注 `【语速：放慢】`

3. **结尾撰写**（关键：最后一句话会被记住最久）：
   - 选择结尾策略：
     - 回扣开场（首尾呼应）
     - 金句收束（给出一句可传播的话）
     - 号召行动（CTA: Call to Action）
     - 愿景画面（描绘美好未来）
   - 结尾标注要求：最后一句前必须有 `【停顿 3s】` + `【语气】`

4. **标注密度控制**：
   - 平均每 100-150 字插入 1 个标注
   - 避免连续 300 字以上无标注（听众会失去节奏感）
   - 开场和结尾标注密度高于主体部分

**写作风格要求**：

| 场景 | 句式特点 | 修辞手法偏好 | 禁忌 |
|------|----------|--------------|------|
| 发布会 | 短句为主，节奏感强 | 排比、对比、设问 | 冗长铺垫、过度谦虚 |
| 年会 | 长短交替，有温度 | 回忆、比喻、排比 | 空洞口号、念流水账 |
| TED 式 | 口语化，像讲故事 | 悬念、画面、转折 | 读稿腔、堆砌术语 |
| 仪式性 | 庄重工整，有韵律 | 引用、对仗、排比 | 过于口语、跑题 |

**输出**：带完整标注的演讲稿正文

---

### Phase 5: 时长估算与终审

**目标**：精确估算演讲时长，进行最终质量审查。

**操作步骤**：

1. **时长计算**：
   - 统计正文总字数（不含标注）
   - 根据演讲类型选择语速基准：
     - 发布会：230 字/分钟（节奏紧凑）
     - 年会致辞：220 字/分钟（从容稳重）
     - TED 式：240 字/分钟（自然口语）
     - 仪式性：200 字/分钟（庄重缓慢）
   - 累加所有停顿标注的秒数
   - 累加互动环节的预估时间（鼓掌约 5-8s，互动提问约 15-30s）
   - 计算总时长

2. **时长调整**（如与目标时长偏差超过 10%）：
   - 超时：标注可删减的段落（标记为 `[可选段落]`），提供精简建议
   - 不足：标注可扩展的位置，建议增加故事/案例/互动

3. **修辞三角检查**：
   - Ethos：是否在开场 1 分钟内建立了可信度？
   - Pathos：情感弧线是否完整？高潮点是否在全篇 2/3 处？
   - Logos：核心论点是否清晰？论据是否充分？

4. **标注质量检查**：
   - 开场是否有语气+停顿标注？
   - 段落过渡处是否有停顿标注？
   - 情感转折处是否有语气切换标注？
   - 关键数据/金句前是否有重音/语速标注？
   - 结尾是否有停顿+语气标注？

**输出**：

```
📊 时长估算报告
- 正文字数：XXX 字
- 语速基准：XXX 字/分钟
- 正文朗读时长：X 分 X 秒
- 停顿累计：X 秒（共 X 处）
- 互动预留：X 秒（共 X 处）
- 预估总时长：X 分 X 秒
- 目标时长：X 分钟
- 偏差：[±X%，符合要求 / 需要调整]

✅ 修辞三角检查
- Ethos（可信度）：[通过/需优化] — [简要说明]
- Pathos（情感）：[通过/需优化] — [简要说明]
- Logos（逻辑）：[通过/需优化] — [简要说明]

✅ 标注完整性检查
- 标注总数：X 个
- 标注密度：约每 XXX 字 1 个标注
- 开场标注：[完整/缺失]
- 过渡标注：[完整/缺失]
- 结尾标注：[完整/缺失]
```

## 流程控制规则

### 交互模式选择

| 用户输入 | 模式 | 行为 |
|----------|------|------|
| 只有场景和主题（< 50 字） | **引导模式** | 执行 Phase 1 提问，等用户回答后继续 |
| 有一定细节（场景+受众+目的） | **半自动模式** | 提 2-3 个关键问题，同时开始修辞分析 |
| 详细 brief（> 200 字） | **全自动模式** | 直接进入 Phase 2，跳过澄清 |
| 用户说"直接写/不用问" | **快速模式** | 基于合理假设直接输出完整演讲稿 |

### 迭代优化

如果用户对演讲稿有反馈：

1. **改内容**：回到 Phase 4 修改对应段落，级联更新时长估算
2. **改结构**：回到 Phase 3 调整结构，重新撰写受影响段落
3. **改策略**：回到 Phase 2 调整修辞三角策略，重新走 Phase 3-5
4. **改场景**：回到 Phase 1 重新澄清，全流程重走

所有修改后必须重新执行 Phase 5 的时长估算和质量检查。

## 参考方法论

- **Aristotle's Rhetorical Triangle**（亚里士多德修辞三角）：Ethos（可信度）、Pathos（情感）、Logos（逻辑）三维说服框架
- **Monroe's Motivated Sequence**（Monroe 动机序列）：Attention → Need → Satisfaction → Visualization → Action
- **Hero's Journey**（英雄之旅叙事结构）：Joseph Campbell 的故事弧线，适用于 TED 式演讲
- **Rule of Three**（三的法则）：演讲中三个并列要点最易被记住
- **Peak-End Rule**（峰终定律）：听众主要记住情感高峰和结尾，指导标注密度分配
