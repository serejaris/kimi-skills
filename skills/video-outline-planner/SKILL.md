# B站视频大纲生成器 — 选题分析 + 结构设计 + 互动设计

为 B 站知识区、生活区、科技区 UP 主生成完整的视频策划大纲，覆盖从选题分析到互动设计的全流程，帮助创作者产出高完播率、高互动率的优质内容。

## Quick Start

用户只需说明：
1. **分区类型**：知识区 / 生活区 / 科技区
2. **视频主题**：如"深度解读芯片制造工艺"、"独居租房改造"、"iPhone vs Android 对比评测"
3. **目标时长**（可选）：默认 8-15 分钟
4. **目标受众**（可选）：如"大学生"、"数码爱好者"、"租房族"
5. **内容调性**（可选）：严肃专业 / 轻松幽默 / 情感共鸣

示例：
> "帮我做一个 B 站知识区的视频大纲，主题是'为什么中国高铁能跑这么快'，时长 10 分钟左右，面向理工科大学生，风格偏硬核科普。"

## 一、选题分析框架

### 1.1 选题四维评估模型

对用户给出的主题，从四个维度进行打分评估（每项 1-5 分）：

| 维度 | 评估要点 | 参考信号 |
|------|---------|---------|
| 搜索需求 | 用户是否在主动搜索该话题 | B 站搜索联想词、百度指数趋势 |
| 话题热度 | 当前社会关注度是否高 | 热搜榜、新闻事件、平台热门话题 |
| 竞品密度 | 同主题优质视频是否已饱和 | 搜索结果前 20 的播放量和发布时间 |
| 差异化空间 | 是否能做出不同于现有内容的角度 | 现有视频的覆盖盲区、独特素材/经历 |

**评估输出格式**：

```
📊 选题评估：[主题名称]

搜索需求：★★★★☆（4/5）— [一句话理由]
话题热度：★★★☆☆（3/5）— [一句话理由]
竞品密度：★★☆☆☆（2/5，低密度=好机会）— [一句话理由]
差异化空间：★★★★★（5/5）— [一句话理由]

综合推荐指数：★★★★☆
选题建议：[采纳 / 调整角度 / 建议更换]
```

### 1.2 三大分区选题策略

#### 知识区选题策略

| 选题类型 | 说明 | 示例 |
|---------|------|------|
| 热点解读型 | 借社会热点切入知识讲解 | "从日本核废水事件聊聊氚的化学" |
| 认知颠覆型 | 打破常识认知，制造信息差 | "你以为的XX其实完全是错的" |
| 系统梳理型 | 对复杂知识做体系化整理 | "一口气看完芯片发展史" |
| 实用方法型 | 提供可直接应用的知识方法 | "学会这个公式，股票估值不再难" |

#### 生活区选题策略

| 选题类型 | 说明 | 示例 |
|---------|------|------|
| 过程记录型 | 记录某件事从头到尾的过程 | "花3000块改造出租屋全过程" |
| 体验分享型 | 亲身体验某件事并分享感受 | "在上海月薪5000是怎样的体验" |
| 挑战实验型 | 设定一个挑战并记录结果 | "连续30天5点起床会发生什么" |
| 清单推荐型 | 整理实用好物/经验清单 | "租房三年踩过的坑和省钱攻略" |

#### 科技区选题策略

| 选题类型 | 说明 | 示例 |
|---------|------|------|
| 产品评测型 | 深度测试某款产品并给出结论 | "M4 MacBook Pro 一个月真实体验" |
| 对比横评型 | 多款产品同场景对比 | "5款千元TWS耳机盲听对比" |
| 技术解析型 | 用通俗语言讲解技术原理 | "5nm芯片到底意味着什么" |
| 趋势预测型 | 基于行业动态分析未来走向 | "2026年手机行业三大趋势预判" |

### 1.3 标题优化公式

根据 B 站算法偏好，标题应遵循以下结构：

| 公式 | 模板 | 示例 |
|------|------|------|
| 数字 + 悬念 | "X个[领域]的[反常识事实]" | "3个你不知道的物理冷知识" |
| 痛点 + 方案 | "[痛点描述]？[方案暗示]" | "租房总被坑？这5条经验帮你省2万" |
| 对比 + 结论 | "[A] vs [B]，[结论暗示]" | "华为 vs 苹果，差距比你想的大" |
| 时间 + 结果 | "[时间段][行为]，[结果]" | "坚持早起100天后，我的变化" |

## 二、结构设计系统

### 2.1 B 站视频黄金结构

```
┌─────────────────────────────────────────────┐
│  第一部分：开头钩子（前30秒）               │
│  → 目标：阻止划走，建立期待                 │
├─────────────────────────────────────────────┤
│  第二部分：背景铺垫（30秒-2分钟）           │
│  → 目标：建立共识，说明"为什么要看这个"     │
├─────────────────────────────────────────────┤
│  第三部分：核心内容（占总时长60%-70%）       │
│  → 目标：分段递进，每段有小高潮             │
├─────────────────────────────────────────────┤
│  第四部分：总结升华（最后1-2分钟）           │
│  → 目标：信息收束 + 情感共鸣 + 互动引导     │
└─────────────────────────────────────────────┘
```

### 2.2 开头钩子设计（前 30 秒）

开头是 B 站视频完播率的决定性因素。以下是 6 种经过验证的钩子类型：

| 钩子类型 | 说明 | 适用分区 | 示例 |
|---------|------|---------|------|
| 结论前置 | 先给出最震撼的结论 | 知识区、科技区 | "告诉你一个事实：你每天喝的水，年龄可能比太阳还大" |
| 冲突开场 | 展示一个反直觉的冲突 | 知识区 | "教科书说的这个知识，其实是错的" |
| 痛点共鸣 | 直击观众的真实困扰 | 生活区 | "是不是每次搬家都觉得自己的东西多到离谱？" |
| 视觉冲击 | 先放最精彩的画面 | 科技区、生活区 | （改造前后对比画面 / 产品拆解特写） |
| 悬念预告 | 留一个待揭晓的悬念 | 全分区 | "今天测试的结果完全出乎我的意料，尤其是第三款" |
| 数据震撼 | 用一个惊人的数据开场 | 知识区、科技区 | "中国每年浪费的粮食，够2亿人吃一年" |

### 2.3 核心内容分段节奏

核心内容应拆分为 3-5 个段落，每段遵循：

```
每段结构 = 段落标题（字幕提示）
          + 论点/观点（15秒内说清）
          + 论据/素材（案例、数据、画面）
          + 小结/过渡（承上启下一句话）
```

**节奏控制原则**：

| 原则 | 说明 |
|------|------|
| 3 分钟高潮法则 | 每 3 分钟必须有一个信息高潮点（反转、新发现、情感点） |
| 信息密度递增 | 前半段铺垫积累，后半段加速释放 |
| 视觉节奏变化 | 每 60-90 秒切换一次画面形式（口播→素材→动画→实拍） |
| 段间过渡 | 每段结尾用一句话勾起下段兴趣："但真正让我震惊的是接下来这个……" |

### 2.4 三大分区结构模板

#### 知识区结构模板（10-15 分钟）

```
00:00-00:30  钩子：抛出反直觉结论 / 惊人数据
00:30-02:00  背景：为什么这个问题值得关注 + 常见误区
02:00-04:00  第一层：基础概念讲清楚（是什么）
04:00-07:00  第二层：深层原因 / 机制分析（为什么）
07:00-10:00  第三层：实际影响 / 应用场景（怎么用）
10:00-12:00  第四层（可选）：前沿进展 / 争议观点
12:00-13:00  总结：核心要点回顾 + 个人观点 + 互动引导
```

#### 生活区结构模板（8-12 分钟）

```
00:00-00:30  钩子：结果展示 / 痛点共鸣 / 悬念预告
00:30-01:30  起因：为什么做这件事（动机和背景）
01:30-03:00  准备阶段：做了哪些准备 + 遇到的第一个问题
03:00-06:00  执行过程：核心经历，按时间线或步骤推进
06:00-08:00  结果展示：最终效果 + 与预期的对比
08:00-09:00  复盘反思：花了多少 + 踩了哪些坑 + 经验建议
09:00-10:00  总结：感悟分享 + 互动引导
```

#### 科技区结构模板（10-15 分钟）

```
00:00-00:30  钩子：最惊艳的测试结果 / 核心结论前置
00:30-02:00  产品/技术概述：参数、定位、价格、竞品对比背景
02:00-05:00  测试一：性能/核心功能（数据+主观感受）
05:00-08:00  测试二：特色功能/使用场景（实际演示）
08:00-10:00  测试三：短板/缺点（客观呈现，不回避问题）
10:00-12:00  横向对比：与竞品的核心差异点
12:00-13:00  购买建议：适合谁 / 不适合谁 + 互动引导
```

### 2.5 结尾设计

结尾需同时完成三个任务：

| 任务 | 做法 |
|------|------|
| 信息收束 | 用 2-3 句话总结全片核心观点，强化记忆 |
| 情感升华 | 从具体内容上升到更宏观的思考或感悟 |
| 行动引导 | 自然过渡到互动引导（详见第三部分） |

## 三、互动设计体系

### 3.1 弹幕互动设计

弹幕是 B 站特有的实时互动形式，需要在大纲中预设弹幕触发点：

| 触发策略 | 说明 | 示例话术 | 推荐时机 |
|---------|------|---------|---------|
| 投票式 | 让观众用弹幕表态 | "觉得A好的打1，觉得B好的打2" | 对比环节 / 观点分歧处 |
| 填空式 | 让观众补充信息 | "你们城市的房租大概是多少？弹幕告诉我" | 涉及地域/个人差异的话题 |
| 猜测式 | 先提问再揭晓答案 | "大家猜猜这个结果是多少？……答案是……" | 测试/实验揭晓前 |
| 共鸣式 | 引发集体认同 | "有没有同款经历的？弹幕飘一波" | 讲到普遍痛点时 |
| 彩蛋式 | 奖励坚持看到此处的观众 | "看到这里的都是真爱粉，弹幕打个'坚持'" | 视频中后段 |

**弹幕设计原则**：
- 每 3-4 分钟设置一个弹幕互动点
- 指令要简短明确（打数字比打文字门槛低）
- 弹幕问题应该有争议性或趣味性，避免只有唯一答案

### 3.2 评论区运营设计

在大纲中预设评论区话题，提升评论数和互动率：

| 策略 | 做法 | 示例 |
|------|------|------|
| 开放式提问 | 在结尾抛出一个没有标准答案的问题 | "你觉得未来5年AI会取代哪些职业？评论区聊聊" |
| 经验征集 | 邀请观众分享自己的经历 | "你们租房时踩过最大的坑是什么？评论区交流一下" |
| 选题征集 | 让观众决定下期内容 | "接下来想看我测哪款产品？评论区投票" |
| 补充纠错 | 主动邀请专业观众补充 | "如果有更了解这个领域的朋友，欢迎在评论区补充" |
| 置顶评论 | 预设一条置顶评论的内容 | 补充视频中提到的资源链接 / 勘误 / 时间轴目录 |

### 3.3 一键三连引导设计

一键三连（点赞+投币+收藏）是 B 站核心互动指标。引导话术要自然不尴尬：

| 引导方式 | 示例话术 | 适用场景 |
|---------|---------|---------|
| 价值交换型 | "如果这个视频帮你省了时间/避了坑，一键三连就是最好的支持" | 干货类内容 |
| 数据里程碑型 | "这个视频做了XX天，如果觉得有收获，三连支持一下" | 高制作成本内容 |
| 轻松玩梗型 | "都看到这了还不三连？你的良心不会痛吗（笑）" | 轻松风格视频 |
| 条件触发型 | "如果你猜对了结果，那就三连一下奖励自己" | 有悬念/猜测环节的视频 |
| 系列订阅型 | "三连+关注，下期更新不迷路" | 系列内容 |

**三连引导原则**：
- 放在结尾总结之后、片尾之前
- 时长控制在 10-15 秒，不要反复强调
- 话术风格要与视频整体调性一致

### 3.4 分 P 与合集设计

对于较长或系列化内容的规划建议：

| 场景 | 建议 |
|------|------|
| 单个主题超过 20 分钟 | 拆分为上/下两 P，每 P 保持完整性 |
| 同一主题多角度 | 规划为合集，每期聚焦一个子话题 |
| 连续更新型内容 | 建立合集，统一封面风格和标题格式 |

## 五、Agent 行为指南

### 5.1 工作流程

当用户要求生成 B 站视频大纲时，按以下步骤执行：

1. **确认需求**：
   - 分区类型（必选）
   - 视频主题（必选）
   - 目标时长（默认 10 分钟）
   - 目标受众（默认该分区主力受众）
   - 内容调性（默认根据分区匹配）

2. **选题分析**：
   - 对主题进行四维评估
   - 给出差异化切入角度建议
   - 生成 3 个标题备选

3. **结构设计**：
   - 根据分区选择对应结构模板
   - 设计开头钩子（至少给出 2 种方案供选择）
   - 规划核心内容的分段和节奏
   - 设计结尾升华

4. **互动设计**：
   - 在每个核心段落中嵌入弹幕互动点
   - 设计评论区话题
   - 编写三连引导话术

5. **输出完整大纲**：
   - 按第四部分的模板格式输出
   - 附上制作备忘

### 5.2 注意事项

- 所有建议基于 B 站平台特性和用户行为习惯，不套用其他平台的逻辑
- 弹幕互动设计是 B 站独有优势，每个大纲必须包含至少 3 个弹幕触发点
- 标题不使用夸大虚假的表述，避免被平台降权
- 选题分析应基于通用的内容策略判断，不依赖实时数据接口
- 如用户未指定分区，根据主题内容自动推荐最匹配的分区
- 如主题过于宽泛，先建议收窄角度再生成大纲
