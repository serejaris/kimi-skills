# 定价页面实操手册

SaaS 定价页面的设计规范、文案框架和转化策略。

---

## 定价页面的核心使命

一个任务：让合适的客户点击合适套餐的行动按钮。页面上的每个元素都应该服务于这个目标，否则就该删掉。

访客来到定价页面时，已经对你的产品感兴趣了。他们现在要回答的问题是：
1. "哪个套餐适合我？"
2. "这个价格值不值？"
3. "有没有什么坑？"

你的页面按照这个顺序回答这三个问题。

---

## 页面结构（滚动顺序）

### 首屏

**计费周期切换（月付/年付）**
- 如果你更希望客户选年付，默认显示年付（大多数转化发生在这里）
- 清晰展示优惠："节省 20%" 的标签，而不只是让客户自己算
- 将切换按钮放在套餐卡片上方

**套餐卡片（三列）**
```
┌─────────────┬─────────────┬─────────────┐
│   Starter   │    Pro      │ Enterprise  │
│             │ ★ Popular   │             │
│   $29/mo    │   $99/mo    │  Custom     │
│             │             │             │
│ For small   │ For growing │ For teams   │
│ teams       │ teams       │ needing     │
│             │             │ control     │
│ • Feature   │ • Feature   │ • Feature   │
│ • Feature   │ • Feature   │ • Feature   │
│ • Feature   │ • Feature   │ • Feature   │
│             │             │             │
│ [Start free]│[Start free] │[Contact us] │
└─────────────┴─────────────┴─────────────┘
```

**每张套餐卡片必须包含：**
- 套餐名称（面向客户群体命名，而不只是"基础/专业"）
- 价格（含计费周期和每席位标注，如适用）
- 一句话定位描述（"适合需要 XX 的成长团队"）
- 4-6 个差异化功能点（这个层级能获得什么）
- 行动按钮（清晰、面向行动——不只是"注册"）
- 中间套餐标注"最受欢迎" / "推荐"

### 首屏以下

**完整功能对比表**
- 列出所有功能
- 按类别分组：核心功能、协作、分析、管理、支持
- 用 ✅ / ❌ 或对勾/横杠标注——不要用模棱两可的文字
- 表头固定，滚动时套餐名称始终可见
- 确保可快速扫读，不要变成文字墙

**客户背书区域**
- 尽量为每个套餐提供 3 条相关的客户评价
- 展示知名客户的 logo
- 如果有真实数据："已服务超过 10,000 个团队"

**常见问题区域（5-7 个问题）**

必须包含的问题：
1. "可以随时取消吗？" → 可以。在设置中取消即可，无需打电话。
2. "试用期结束后会怎样？" → 我们会询问您是否继续使用。
3. "可以切换套餐吗？" → 可以，随时升级或降级，按比例计费。
4. "支持哪些付款方式？" → 信用卡；企业年付可开发票。
5. "我的数据安全吗？" → SOC 2 Type II / ISO 27001 / 简要安全声明。
6. "如果顶级套餐还不够用呢？" → 联系我们：[企业咨询表单链接]。

**企业行动号召区域**
- 在套餐卡片下方单独设置一行或一个区块
- "需要定制方案或产品演示？" → [联系销售] 按钮
- 说明适用对象：超过 X 席位的团队、有特定合规需求、需要定制合同

## 文案框架

### 套餐命名

尽量避免通用名称。带身份感的命名比纯价格层级更有效。

| 通用命名 | 更好的命名 | 原因 |
|---------|-----------|------|
| 免费 / 基础 / 专业 | 个人版 / 工作室版 / 机构版 | 对应客户群体 |
| 入门 / 成长 / 企业 | 开发者 / 团队 / 商业 | 对应使用场景 |
| 个人 / 团队 / 组织 | 创作者 / 协作者 / 企业 | 对应角色 |

如果你的用户类别确实比较模糊，就用简单的名称。不要为了创意而强加让人困惑的命名。

### 行动按钮文案

根据具体情况匹配文案：

| 场景 | 按钮文案 |
|------|---------|
| 有免费试用 | "开始免费试用" |
| 免费增值模式 | "免费开始" |
| 无试用，直接购买 | "获取[套餐名]" |
| 企业版 / 联系销售 | "联系我们" 或 "预约演示" |
| 年付、高价格 | "预约沟通" |

避免：
- "注册" —— 太通用，没有价值感
- "订阅" —— 听起来像订阅新闻简报
- "立即购买" —— 太交易化，缺乏利益导向
- "了解更多" —— 在定价页面上，这是一条死胡同

### 价格展示

| 场景 | 展示方式 |
|------|---------|
| 月付价格 | "$99/月" |
| 年付，按月显示 | "$83/月，按年计费" |
| 年付，一次性支付 | "$996/年" 附"折合每月 XX" |
| 按席位 | "$15/用户/月" |
| 按用量 | "低至 $0.002/次调用" |
| 企业定制 | "定制报价" 或 "起步价 $X" |

年付优惠始终以百分比或金额展示（选视觉效果更大的那个）。

## 转化策略

### 锚定效应

**价格锚定：** 用户看到的第一个数字会建立参考框架。如果先展示 $500/月的套餐，$99 就会显得很便宜。

如果你想推动用户选择中间套餐：
- 从左到右排列：高级 → 专业（推荐）→ 入门
- 或者通过视觉处理突出中间套餐（更大的卡片、边框、颜色）
- 用户的视线会被视觉上最突出的选项吸引

### "推荐"标识

不要只是给中间套餐加个文字标签，要让它在视觉上非常明显：
- 深色背景或品牌色
- 略高的卡片
- "最受欢迎" 或 "推荐大多数团队使用" 的标签
- 在 Tab 顺序中排第一个行动按钮

### 年付切换默认值

研究一致表明，默认展示年付价格能提升年付选择率。展示切换按钮，但默认选中年付。

如果你更需要月付客户（用于现金流测试或低承诺产品），则默认选中月付。

### 定价页面 SEO 注意事项

定价页面通常会在"[公司名] 定价"这类搜索词中获得排名。重要原因：
- 竞品可能在你的品牌定价关键词上投放广告
- 页面需要加载快速、结构清晰
- 在结构化数据中包含定价信息（JSON-LD Schema: PriceSpecification）

## 定价页面审查清单

每项评分 0-2（0 = 缺失，1 = 有但较弱，2 = 做得好）：

**首屏**
- [ ] 计费周期切换可见
- [ ] 年付优惠清晰展示
- [ ] 三个套餐卡片差异化明显
- [ ] "最受欢迎"/推荐套餐突出显示
- [ ] 每个套餐有行动按钮

**内容**
- [ ] 完整功能对比表
- [ ] 常见问题区域（5 个以上）
- [ ] 客户背书 / logo
- [ ] 企业级行动号召

**文案**
- [ ] 套餐名称有实际意义（不只是"基础/专业"）
- [ ] 价格表达清晰无歧义（每用户？每月？如何计费？）
- [ ] 行动按钮面向行动
- [ ] 每个套餐有一句话定位

**信任**
- [ ] 安全认证标识（B2B 场景）
- [ ] 退款保证或取消政策可见
- [ ] 明确标注"随时取消"

**评分解读：**
- 22-24 分：页面优秀。可以测试具体元素优化。
- 16-21 分：基础良好。修复薄弱环节。
- <16 分：存在重大缺失。按照本手册重建。

## 定价页面 A/B 测试建议

**高影响、易测试：**
1. 默认计费周期切换（年付 vs 月付）
2. "最受欢迎" 标识位置
3. 行动按钮文案（"开始免费试用" vs "获取专业版"）
4. 价格展示方式（每月 vs 每年）

**中等影响、需要更多准备：**
5. 套餐名称策略（按客户群体 vs 按功能）
6. 首屏卡片展示的功能数量（3 个 vs 6 个）
7. 客户背书位置（首屏上方 vs 下方）
8. FAQ 展开方式（手风琴 vs 全部展开）

**高影响、执行难度大：**
9. 实际价格点（达到统计显著性需要更长时间）
10. 三级套餐 vs 两级套餐
11. 增加 vs 移除免费层级

**定价测试最低流量要求：** 每个变体每周 500+ 访客。低于此数，结果不具统计意义。
