# Exam Question Generator — 按布鲁姆认知层次生成模拟题

基于布鲁姆认知分类学（Bloom's Taxonomy），为任意学科/主题生成多题型模拟题（选择题、简答题、案例分析题），每题附详细解析，覆盖从低阶到高阶的完整认知层次。

## Quick Start

用户只需说明：
1. **学科/主题**：如"微观经济学-供需理论"、"数据结构-二叉树"、"项目管理-风险管理"
2. **认知层次**（可选）：记忆 / 理解 / 应用 / 分析 / 评价 / 创造，或"全部层次"
3. **题型**（可选）：选择题 / 简答题 / 案例分析题，或"混合题型"
4. **题量**（可选）：默认每个层次 1 题
5. **难度偏好**（可选）：基础 / 中等 / 进阶

示例：
> "帮我针对'市场营销-4P理论'出一套模拟题，覆盖布鲁姆六个层次，混合题型，每层1题。"

## 一、布鲁姆认知分类学参考框架

### 1.1 六级认知层次总览

| 层次 | 英文 | 核心动作 | 典型动词 | 适配题型 |
|------|------|---------|---------|---------|
| L1 记忆 | Remember | 识别、回忆事实性知识 | 列举、定义、识别、匹配 | 选择题 |
| L2 理解 | Understand | 解释、归纳、推断含义 | 解释、举例、分类、比较、概括 | 选择题、简答题 |
| L3 应用 | Apply | 在新情境中使用知识 | 计算、执行、使用、演示 | 选择题、简答题 |
| L4 分析 | Analyze | 分解信息、发现关系和结构 | 区分、组织、归因、比较差异 | 简答题、案例分析题 |
| L5 评价 | Evaluate | 基于准则做出判断 | 评估、批判、论证、辩护 | 简答题、案例分析题 |
| L6 创造 | Create | 整合要素形成新方案 | 设计、构建、规划、提出 | 案例分析题 |

### 1.2 层次 × 题型匹配矩阵

| 题型 \ 层次 | L1 记忆 | L2 理解 | L3 应用 | L4 分析 | L5 评价 | L6 创造 |
|------------|---------|---------|---------|---------|---------|---------|
| 选择题     | ★★★   | ★★★   | ★★    | ★      | —      | —      |
| 简答题     | ★      | ★★    | ★★★   | ★★★   | ★★    | ★      |
| 案例分析题  | —      | ★      | ★★    | ★★★   | ★★★   | ★★★   |

> ★★★ = 最佳匹配，★★ = 适用，★ = 可用但非最优，— = 不推荐

## 二、题型规范与模板

### 2.1 选择题规范

**结构**：
```
题号. [L层次标签] 题干
A. 选项A
B. 选项B
C. 选项C
D. 选项D

✅ 正确答案：X
📝 解析：
- 考点：…
- 正确选项分析：…
- 干扰项分析：逐一说明其他选项为何错误
- 认知层次说明：本题考查的是…层次的…能力
```

**编写规则**：
- 题干表述清晰完整，避免否定句式（如确需否定，加粗"不"字）
- 四个选项长度相近，避免最长选项总是正确答案
- 干扰项须有合理性（常见误解、易混概念），不设明显荒谬选项
- 选项间互斥，不出现"以上皆是/皆非"

### 2.2 简答题规范

**结构**：
```
题号. [L层次标签] 题干（明确作答要求和预期篇幅）

📋 参考答案：
（结构化的要点式回答）

📝 解析：
- 考点：…
- 得分要点：列出关键得分点及分值分配建议
- 常见失分点：学生容易遗漏或答偏的方向
- 认知层次说明：本题考查的是…层次的…能力
```

**编写规则**：
- 题干明确要求"简述"、"比较"、"说明"等，暗示作答方向
- 参考答案须结构化（分点/分步），便于评分
- 提供得分要点和分值建议

### 2.3 案例分析题规范

**结构**：
```
题号. [L层次标签] 案例背景
（200-400字的情境描述，包含足够的决策信息和适量干扰信息）

问题：
(1) …（通常2-3个递进式子问题）
(2) …
(3) …

📋 参考答案：
(1) …
(2) …
(3) …

📝 解析：
- 考点：…
- 案例中的关键信息提取：标注哪些信息是解题关键，哪些是干扰
- 分析框架/模型建议：推荐适用的理论工具
- 得分要点与评分建议
- 认知层次说明：各子问题分别对应的认知层次
```

**编写规则**：
- 案例情境须真实合理（可虚构公司名/人名，但行业逻辑须成立）
- 子问题按认知层次递进设计（如：先分析现状 → 再评价方案 → 最后提出建议）
- 案例中混入适量非关键信息，训练信息筛选能力
- 不在案例描述中暗示标准答案

## 三、解析编写标准

每道题的解析必须包含以下要素：

| 要素 | 说明 | 必选/可选 |
|------|------|----------|
| 考点 | 本题考查的核心知识点 | 必选 |
| 正确答案分析 | 为何正确、依据什么原理 | 必选 |
| 错误项/失分点分析 | 常见错误及其原因 | 必选 |
| 认知层次说明 | 本题对应布鲁姆哪个层次，考查什么能力 | 必选 |
| 关联知识 | 与本题相关的延伸知识点 | 可选 |
| 解题技巧 | 实用的答题方法或记忆诀窍 | 可选 |

## 四、难度校准指南

### 4.1 三级难度标准

| 难度 | 标签 | 特征 | 预期正确率 |
|------|------|------|-----------|
| 基础 | ⭐ | 直接考查教材核心概念，无需推理 | 70%-90% |
| 中等 | ⭐⭐ | 需要理解后应用，或涉及概念辨析 | 40%-70% |
| 进阶 | ⭐⭐⭐ | 需综合多知识点，或在复杂情境中判断 | 20%-40% |

### 4.2 认知层次与难度的关系

- L1-L2 题目通常为基础或中等难度
- L3-L4 题目通常为中等或进阶难度
- L5-L6 题目通常为进阶难度
- 但高层次不一定等于高难度：一道简单情境的"评价"题可能比一道复杂计算的"应用"题更容易

## 五、完整出题示例

### 示例主题：项目管理 — 风险管理

---

**1. [L1 记忆] 选择题 ⭐**

在项目风险管理中，"风险登记册"（Risk Register）的主要用途是什么？

A. 记录项目的财务预算和支出明细
B. 系统化记录已识别的风险及其应对措施
C. 跟踪项目团队成员的绩效考核
D. 存储项目的合同和法律文件

✅ 正确答案：B
📝 解析：
- 考点：风险登记册的定义与用途
- 正确选项分析：风险登记册是记录已识别风险、风险分析结果、应对策略和监控状态的核心文件，贯穿风险管理全过程
- 干扰项分析：A 是预算管理工具的功能；C 属于人力资源管理范畴；D 属于采购管理或合同管理范畴
- 认知层次说明：本题考查 L1 记忆层次，要求识别和回忆风险管理基本概念

---

**2. [L2 理解] 选择题 ⭐**

以下哪个例子最准确地体现了"风险转移"策略？

A. 项目经理决定不开展某项高风险活动
B. 项目团队为关键设备购买保险
C. 项目团队预留了额外的时间缓冲
D. 项目经理选择接受风险并持续监控

✅ 正确答案：B
📝 解析：
- 考点：风险应对策略的辨析（回避、转移、减轻、接受）
- 正确选项分析：购买保险将风险的财务后果转移给保险公司，是典型的风险转移
- 干扰项分析：A 是风险回避（不做这件事）；C 是风险减轻（降低影响）；D 是风险接受（主动承担）
- 认知层次说明：本题考查 L2 理解层次，要求对风险应对策略进行分类和举例识别

---

**3. [L3 应用] 简答题 ⭐⭐**

某软件开发项目中，团队识别出一个风险："核心开发人员可能在项目中期离职"。请运用定性风险分析方法，评估该风险的优先级，并提出至少两种应对措施。

📋 参考答案：
1. **定性评估**：
   - 发生概率：中等（行业人员流动率约15%-20%）
   - 影响程度：高（核心开发人员掌握关键技术知识，离职将导致进度延迟和知识流失）
   - 优先级：高（概率中 × 影响高 = 需优先应对）

2. **应对措施**：
   - 减轻策略：建立代码文档化和知识共享机制，确保关键知识不集中在一人身上
   - 减轻策略：交叉培训团队成员，至少两人掌握每个核心模块
   - 转移策略：与外包公司签订备用人力协议，关键时刻可快速补充
   - 接受策略（主动）：预留10%-15%的进度缓冲，以应对可能的人员变动

📝 解析：
- 考点：定性风险分析（概率-影响矩阵）+ 风险应对策略
- 得分要点：①正确评估概率和影响（2分）②优先级判断合理（1分）③至少两种应对措施且分类正确（4分）④措施具体可执行（3分）
- 常见失分点：仅给出笼统的"加强管理"，未给出具体可执行的措施；混淆应对策略类型
- 认知层次说明：本题考查 L3 应用层次，要求将风险分析方法应用于具体项目情境

**4. [L4 分析] 案例分析题 ⭐⭐**

**案例背景：**

华创科技公司正在开发一款新型智能家居控制系统，项目周期12个月，预算500万元。项目进行到第4个月时，项目经理发现以下情况：

- 硬件供应商通知芯片交期将延迟6周（原因：上游原材料短缺）
- 软件团队提出需要新增一个AI语音识别模块（客户新需求），预计增加80万元成本和2个月工期
- 公司刚宣布年底前要削减10%的研发预算
- 竞争对手预计在8个月后发布类似产品
- 测试团队主管刚提交了离职申请，交接期仅2周

**问题：**
(1) 请识别案例中的所有风险，并按"概率-影响"进行分类排序
(2) 分析这些风险之间的关联性和可能的连锁效应
(3) 为排名前三的风险各提出一套应对方案

📋 参考答案：

(1) **风险识别与排序**：

| 排序 | 风险 | 概率 | 影响 | 优先级 |
|------|------|------|------|--------|
| 1 | 芯片交期延迟6周 | 高（已确认） | 高（阻塞硬件开发） | 极高 |
| 2 | 测试主管离职致知识流失 | 高（已确认） | 高（影响质量保障） | 极高 |
| 3 | 预算削减10%与新需求增加80万的矛盾 | 高 | 高（项目范围/预算冲突） | 极高 |
| 4 | 新增AI模块导致工期延长 | 中 | 高（可能错过市场窗口） | 高 |
| 5 | 竞争对手8个月后发布同类产品 | 中 | 中（市场先发优势受威胁） | 中 |

(2) **关联性分析**：
- 芯片延迟 + AI模块增加 → 双重工期压力，总延迟可能超过2个月
- 预算削减 + 成本增加 → 资源缺口加剧，可能被迫砍功能
- 测试主管离职 + 工期压缩 → 质量风险在项目后期集中爆发
- 如果项目延期超过3个月，将晚于竞品上市，商业价值大幅下降

(3) **前三风险应对方案**：

**风险1 — 芯片交期延迟6周**：
- 减轻：立即联系备选供应商询价，启动双源采购策略
- 转移：与当前供应商协商延迟违约条款，将部分损失转移至供应商
- 接受（主动）：调整项目计划，将软件模块开发前置，硬件集成后移，利用等待期消化其他任务
- 时间节点：1周内完成备选供应商评估，2周内确定最终供货方案

**风险2 — 测试主管离职致知识流失**：
- 减轻：在2周交接期内完成核心测试用例、测试环境配置、已知缺陷清单的文档化交接
- 减轻：安排团队成员与测试主管结对工作，进行知识转移
- 转移：快速启动外部测试团队招聘或外包，设定"到岗即可接手"的交接标准
- 时间节点：第1周完成知识文档化，第2周完成结对培训，第4周前完成补员

**风险3 — 预算削减与新需求成本矛盾**：
- 回避：与客户协商AI语音模块的优先级，评估是否可纳入二期交付而非一期
- 减轻：对新需求进行最小可行范围（MVP）评估，在功能和成本间寻找折中方案
- 升级：准备成本影响分析报告，提请管理层在"削减预算"与"满足客户新需求"间做出优先级决策
- 时间节点：1周内完成MVP方案和成本分析，2周内获得管理层决策

📝 解析：
- 考点：风险识别、定性风险分析、风险关联性分析
- 案例关键信息：5个明确的风险事件，存在时间和资源上的关联
- 分析框架建议：概率-影响矩阵、风险关联图
- 得分要点：①完整识别5个风险（5分）②排序合理有依据（3分）③关联性分析逻辑清晰（4分）④应对方案具体可行（6分）
- 认知层次说明：子问题(1)为 L4 分析（分解和分类），子问题(2)为 L4 分析（发现关联），子问题(3)为 L5 评价/L6 创造（综合判断并提出方案）

**5. [L5 评价] 简答题 ⭐⭐⭐**

有人主张"敏捷项目不需要正式的风险管理流程，因为短迭代周期本身就是最好的风险管理"。请评价这一观点，说明你的理由。

📋 参考答案：

这一观点部分合理但不全面：

**合理之处**：
- 短迭代（1-4周）确实能更早暴露问题，缩短风险潜伏期
- 持续交付和频繁反馈机制有助于快速调整方向
- Sprint回顾会议提供了定期审视风险的机会

**不足之处**：
- 短迭代能应对战术层面的风险，但对战略性风险（如技术选型失误、关键供应商依赖）缺乏预见性
- 某些风险需要提前数月准备（如合规认证、硬件采购长交期），迭代周期内无法消化
- 团队可能陷入"救火模式"，只处理眼前问题而忽视中长期风险

**结论**：
敏捷的短迭代是有效的风险缓解手段，但不能替代系统性的风险识别和前瞻性规划。最佳实践是将轻量化的风险管理流程嵌入敏捷框架中。

📝 解析：
- 考点：敏捷方法论与传统风险管理的关系
- 得分要点：①能识别观点中的合理部分（3分）②能指出不足（3分）③给出平衡的结论而非非黑即白（2分）④论述有理有据（2分）
- 常见失分点：完全同意或完全否定，缺乏辩证思维；论述空洞无具体例子
- 认知层次说明：本题考查 L5 评价层次，要求基于专业知识对一个观点进行多角度批判性评估

**6. [L6 创造] 案例分析题 ⭐⭐⭐**

你被任命为一家中型电商公司的项目经理，负责在6个月内完成支付系统从第三方服务迁移至自建系统的项目。公司年交易额约5亿元，支付不可中断。请设计一套完整的风险管理计划框架，至少包含：风险识别方法、评估标准、Top 5 预判风险及应对策略、监控与报告机制。

📋 参考答案：（应包含完整的风险管理计划框架，结构清晰、内容具体、逻辑严密）

📝 解析：
- 考点：风险管理计划的完整设计
- 得分要点：①方法论选择合理（3分）②评估标准明确可操作（3分）③Top 5 风险识别准确且覆盖技术/业务/人员维度（5分）④应对策略具体可执行（5分）⑤监控机制包含频率、指标、升级路径（4分）
- 认知层次说明：本题考查 L6 创造层次，要求综合运用风险管理知识，为真实场景设计完整的管理框架

## 六、Agent 行为指南

### 6.1 出题流程

当用户要求生成模拟题时，按以下步骤执行：

1. **确认需求**：
   - 学科/主题（必选）
   - 认知层次范围（默认：全部六层）
   - 题型偏好（默认：混合题型）
   - 题量（默认：每层 1 题）
   - 难度（默认：中等）

2. **规划题目分布**：
   - 根据层次 × 题型匹配矩阵，为每个层次选择最佳题型
   - 确保题型分布合理（不全是选择题，也不全是案例题）

3. **逐题生成**：
   - 按 L1 → L6 顺序递进出题
   - 每题标注层次标签和难度星级
   - 题干使用与层次匹配的动词（参见 1.1 典型动词）

4. **编写解析**：
   - 按照第三节"解析编写标准"完成每题的解析
   - 解析须覆盖全部必选要素

5. **输出格式**：
   - 开头给出题目总览表（层次/题型/难度/考点）
   - 按层次顺序逐题输出
   - 每题之间用分隔线隔开
   - 最后附上考点分布总结

### 6.3 注意事项

- 题目内容须基于学科的通用知识，不依赖特定教材版本
- 案例中的公司名、人名须为虚构，避免使用真实企业信息
- 选择题正确答案位置须随机分布（不总是 B 或 C）
- 如用户未指定主题的具体范围，先询问确认再出题
- 每道题的认知层次须名副其实：L1 不应要求分析，L6 不应只考记忆
