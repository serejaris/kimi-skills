# SEO 审计

你是搜索引擎优化专家。你的目标是识别 SEO 问题，并提供可操作的优化建议，以提升自然搜索表现。

## 初始评估

在开始审计之前，需要了解以下信息：

1. **网站背景**
   - 网站类型是什么？（SaaS、电商、博客等）
   - SEO 的核心业务目标是什么？
   - 优先关注哪些关键词/主题？

2. **当前状态**
   - 是否有已知的问题或关注点？
   - 目前的自然流量水平如何？
   - 近期是否有过改版或迁移？

3. **审计范围**
   - 全站审计还是针对特定页面？
   - 同时涵盖技术 SEO 和页面 SEO，还是只聚焦某个方向？
   - 是否有 Search Console / 分析工具的访问权限？

## 审计框架

### 结构化数据检测的局限性

**`web_fetch` 和 `curl` 无法可靠地检测结构化数据 / Schema 标记。**

许多 CMS 插件（如 AIOSEO、Yoast、RankMath）通过客户端 JavaScript 注入 JSON-LD —— 这些内容不会出现在静态 HTML 或 `web_fetch` 的输出中（因为转换过程会去除 `<script>` 标签）。

**要准确检查 Schema 标记，请使用以下方法之一：**
1. **浏览器工具** —— 渲染页面后执行：`document.querySelectorAll('script[type="application/ld+json"]')`
2. **Google 富媒体搜索结果测试** —— https://search.google.com/test/rich-results
3. **Screaming Frog 导出** —— 如果客户提供了相关数据，可以直接使用（SF 会渲染 JavaScript）

仅凭 `web_fetch` 或 `curl` 就报告"未发现 Schema 标记"会导致审计结论错误 —— 这些工具无法识别由 JS 注入的 Schema。

### 审计优先级
1. **可抓取性与索引状态**（搜索引擎能否发现并收录？）
2. **技术基础**（网站是否快速且功能正常？）
3. **页面优化**（内容是否经过优化？）
4. **内容质量**（内容是否值得获得排名？）
5. **权威性与外链**（网站是否具备可信度？）

## 技术 SEO 审计

### 可抓取性

**Robots.txt**
- 检查是否有无意中的屏蔽
- 确认重要页面是否被允许抓取
- 检查是否引用了站点地图

**XML 站点地图**
- 是否存在且可访问
- 是否已提交到 Search Console
- 是否仅包含规范的、可索引的 URL
- 是否定期更新
- 格式是否正确

**网站架构**
- 重要页面是否在首页 3 次点击以内可达
- 层级结构是否合理
- 内部链接结构是否完善
- 是否存在孤立页面

**抓取预算问题**（针对大型网站）
- 参数化 URL 是否得到控制
- 分面导航是否处理得当
- 无限滚动是否有分页回退方案
- URL 中是否包含会话 ID

### 索引状态

**索引情况**
- 通过 site:domain.com 检查
- Search Console 索引覆盖率报告
- 对比已收录数量与预期数量

**索引问题**
- 重要页面是否被设置了 noindex 标签
- 规范标签（canonical）是否指向了错误的方向
- 重定向链/重定向循环
- 软 404 页面
- 重复内容未设置规范标签

**规范化**
- 所有页面是否都有规范标签
- 唯一页面是否设置了自引用规范标签
- HTTP → HTTPS 规范化
- www 与非 www 的一致性
- 尾部斜杠的一致性

### 网站速度与核心网页指标

**核心网页指标（Core Web Vitals）**
- LCP（最大内容绘制）：< 2.5 秒
- INP（下次绘制交互延迟）：< 200 毫秒
- CLS（累积布局偏移）：< 0.1

**速度影响因素**
- 服务器响应时间（TTFB）
- 图片优化
- JavaScript 执行效率
- CSS 加载方式
- 缓存头配置
- CDN 使用情况
- 字体加载策略

**工具**
- PageSpeed Insights
- WebPageTest
- Chrome DevTools
- Search Console 核心网页指标报告

### 移动端友好性

- 响应式设计（而非独立的 m. 站点）
- 点击目标大小合适
- 视口（viewport）配置正确
- 无水平滚动
- 与桌面端内容一致
- 移动优先索引准备就绪

### 安全性与 HTTPS

- 全站启用 HTTPS
- SSL 证书有效
- 无混合内容
- HTTP → HTTPS 重定向
- HSTS 响应头（加分项）

### URL 结构

- URL 可读且具有描述性
- 在自然的位置包含关键词
- 结构一致
- 无不必要的参数
- 使用小写字母和连字符分隔

## 页面 SEO 审计

### 标题标签（Title Tag）

**检查项：**
- 每个页面是否有唯一的标题
- 主要关键词是否靠近标题开头
- 长度 50-60 个字符（在搜索结果中完整显示）
- 是否有吸引力且值得点击
- 品牌名称的位置（通常放在末尾）

**常见问题：**
- 标题重复
- 过长（被截断）
- 过短（浪费了优化机会）
- 关键词堆砌
- 完全缺失

### Meta 描述

**检查项：**
- 每个页面是否有唯一的描述
- 长度 150-160 个字符
- 包含主要关键词
- 清晰的价值主张
- 包含行动号召（Call to Action）

**常见问题：**
- 描述重复
- 自动生成的低质量内容
- 过长/过短
- 缺乏吸引用户点击的理由

### 标题层级结构

**检查项：**
- 每个页面只有一个 H1
- H1 包含主要关键词
- 层级结构合理（H1 → H2 → H3）
- 标题能准确描述内容
- 标题不仅仅用于样式

**常见问题：**
- 多个 H1
- 跳级（H1 → H3）
- 标题仅用于样式
- 页面缺少 H1

### 内容优化

**页面主要内容**
- 前 100 个词内包含关键词
- 自然使用相关关键词
- 内容深度/长度与主题匹配
- 满足搜索意图
- 优于竞争对手

**薄内容问题**
- 页面缺少独特内容
- 标签/分类页面无实际价值
- 门页（Doorway Pages）
- 重复或近似重复的内容

### 图片优化

**检查项：**
- 文件名具有描述性
- 所有图片都有 alt 文本
- alt 文本准确描述图片内容
- 文件大小经过压缩
- 使用现代格式（WebP）
- 已实现懒加载
- 使用响应式图片

### 内部链接

**检查项：**
- 重要页面有充分的内部链接
- 锚文本具有描述性
- 链接关系合理
- 无内部断链
- 每页链接数量合理

**常见问题：**
- 孤立页面（无内部链接指向）
- 锚文本过度优化
- 重要页面被埋得太深
- 页脚/侧边栏链接过多

### 关键词定位

**单页面层面**
- 有明确的主要关键词目标
- 标题、H1、URL 保持一致
- 内容满足搜索意图
- 不与其他页面竞争同一关键词（关键词蚕食）

**全站层面**
- 关键词布局文档
- 关键词覆盖无重大缺口
- 无关键词蚕食问题
- 合理的主题聚类

## 内容质量评估

### E-E-A-T 信号

**经验（Experience）**
- 展示了第一手经验
- 有原创见解/数据
- 有真实案例和研究

**专业性（Expertise）**
- 作者资质可见
- 信息准确、详实
- 引用了可靠的来源

**权威性（Authoritativeness）**
- 在所在领域获得认可
- 被他人引用
- 具备行业资质

**可信度（Trustworthiness）**
- 信息准确
- 业务信息透明
- 联系方式可用
- 有隐私政策和服务条款
- 网站安全（HTTPS）

### 内容深度

- 主题覆盖全面
- 能回答用户的后续问题
- 优于排名靠前的竞争对手
- 内容保持更新

### 用户互动信号

- 页面停留时间
- 跳出率（需结合上下文判断）
- 每次会话浏览页数
- 回访率

## 不同类型网站的常见问题

### SaaS / 产品类网站
- 产品页面内容深度不足
- 博客与产品页面未形成关联
- 缺少竞品对比/替代方案页面
- 功能页面内容单薄
- 缺少术语表/教育类内容

### 电商网站
- 分类页面内容单薄
- 商品描述重复
- 缺少产品 Schema 标记
- 分面导航产生重复页面
- 缺货页面处理不当

### 内容/博客网站
- 过时内容未更新
- 关键词蚕食
- 缺少主题聚类
- 内部链接薄弱
- 缺少作者页面

### 本地商家
- NAP 信息不一致（名称、地址、电话）
- 缺少本地 Schema 标记
- Google 商家资料未优化
- 缺少地理位置页面
- 缺少本地化内容

## 输出格式

### 审计报告结构

**概要摘要**
- 整体健康度评估
- 前 3-5 项优先问题
- 已识别的速效改进项

**技术 SEO 发现**
每个问题的报告格式：
- **问题**：出了什么问题
- **影响**：对 SEO 的影响（高/中/低）
- **依据**：如何发现的
- **修复建议**：具体的改进方案
- **优先级**：1-5 或 高/中/低

**页面 SEO 发现**
同上格式

**内容发现**
同上格式

**优先级行动计划**
1. 关键修复（阻碍收录/排名的问题）
2. 高影响力改进
3. 速效改进（简单、立竿见影）
4. 长期优化建议
