# Structure Contract -- Consulting Market Insight Report

## Document Section Hierarchy

The report follows a standard 32-page consulting report architecture, flowing continuously without numbered section dividers:

```
Cover Page (p1)
Author Profiles & Acknowledgments (p2)
Table of Contents (p3)
Executive Summary / 摘要 (p4-7)
  - Introduction paragraph (research methodology, scope)
  - Per-chapter summary paragraphs with data highlights

"Full Report" Photo Divider Page (p8-9) -- full-width photo + "完整报告" title + intro

Full Report Body -- flows directly from divider without numbered breaks
  Chapter 1: Year-in-Review / 回顾2022 (p10-13)
    - Category dynamics (growth by segment)
    - Channel dynamics
    - Brand dynamics (market share shifts)
  Chapter 2: Current Period Market Insights / Q1/Q2 Analysis (p14-26)
    - Category-level updates
    - Price dynamics trends
    - Channel updates and shifts
    - Recovery pattern analysis
  Chapter 3: Future Trends & Recommendations / 未来趋势展望 (p27-30)
    - Trends outlook
    - Implications for brands
    - Implications for retailers

Back Cover (p30-32) -- logo + office list + contact info
```

## Content Flow Pattern

### Executive Summary Pattern (p4-7)
- 1-3 dense text pages summarizing all key findings
- Introduction paragraph: research methodology, tracked categories, data sources
- One paragraph per major chapter/section with specific data points and percentages
- **No charts in the summary section** -- text only
- Cross-references to charts in body: "（见图X）"

### "Full Report" Photo Divider Page Pattern (p8-9)
- Full-width photograph at top (shopping/supermarket scene)
- Large bold title "完整报告" overlaid or just below image
- 1-2 paragraph overview of what the full report covers
- Research scope reminder (tracking years, categories tracked)
- No running header on this page (exception)

### Chapter Heading Pattern (within body pages)
- Chapter heading in red bold (e.g., "回顾2022")
- Introductory context-setting paragraph (macro data, market overview)
- Subsection headings in black bold (e.g., "品类发展动态：包装食品和家庭护理品类引领增长")
- Data paragraphs with chart references

### Analysis Paragraph Pattern

**CRITICAL: The report body must read as flowing prose, NOT as bullet lists, data tables, or labeled blocks (INSIGHT/DRIVER/RECOMMENDATION). Structured analysis frameworks inform the thinking but must be invisible in the final output — the reader sees polished consulting narrative, not analytical scaffolding.**

Each analytical section is written as **connected paragraphs** following this 5-step internal logic (aligned with `analysis_framework.md` Module 6). The 5 steps are fused into natural prose, NOT displayed as labels:

1. **Context statement** -- setting the scene with macro data
2. **Key finding** -- the main insight backed by data (must pass the "So what?" test)
3. **Supporting evidence** -- chart reference (见图X) with specific numbers
4. **Explanation** -- why this trend is happening (root cause attribution)
5. **Implications** -- what it means for stakeholders (actionable recommendation)

Every paragraph must trace back to an analytical framework from `analysis_framework.md`:
- Market sizing claims → Module 1 (top-down / bottom-up cross-validation)
- Category growth analysis → Module 2 (penetration × frequency × spend decomposition)
- Channel commentary → Module 3 (channel share tracking + efficiency comparison)
- Consumer insights → Module 4 (segmentation + need-state analysis)
- Competitive dynamics → Module 5 (share attribution + concentration metrics)

### Consulting Narrative Writing Rules

#### Paragraph Structure
Each analysis paragraph follows the **"assertion → evidence → interpretation → implication"** flow, written as continuous prose:

**Good example** (consulting style):
> 中国移动互联网的增长引擎正在发生结构性转换。从增长分解来看，2025年用户规模仅增长2.6%，但人均使用时长同比增长8.4%至186.2小时/月，创下近年新高（见图2）。这一分化表明，行业竞争的核心已从"争夺新用户"转向"争夺用户时间"。对于平台企业而言，这意味着产品策略应从拉新获客转向提升单用户时长和场景深度——AI应用的爆发正是这一逻辑的最佳注脚：AIGC App全年净增用户超2亿，成为时长增长的核心驱动力。

**Bad example** (data dump / labeled blocks):
> INSIGHT: 时长增速高于用户增速
> DRIVER: AI应用带动
> IMPLICATION: 应转向用户深度运营
> RECOMMENDATION: 投入AI场景

#### Paragraph Transition Patterns

Use these transition devices to connect paragraphs into flowing narrative:

**Deepening** (from surface to root cause):
- "这一现象的背后是..."
- "进一步拆解可以发现..."
- "造成这一分化的核心原因在于..."
- "从增长归因来看..."

**Contrasting** (unexpected finding or counterpoint):
- "然而，值得注意的是..."
- "与此形成鲜明对比的是..."
- "尽管整体趋势向好，但结构性分化正在加剧..."
- "这一趋势并非没有隐忧..."

**Extending** (from one finding to the next):
- "这一趋势在渠道层面同样得到验证..."
- "类似的逻辑也出现在...领域..."
- "更为深远的影响在于..."
- "如果将视角从...扩展到...，可以看到..."

**Concluding** (from analysis to implication):
- "对于品牌而言，这意味着..."
- "这一格局变化的战略含义是..."
- "综合以上分析，我们认为..."
- "展望未来，这一趋势的延续将..."

#### Section Opening Pattern
Each chapter opens with a **context-setting paragraph** that frames the section's core question:

**Template**:
> [宏观背景1-2句] + [核心发现/论点1句，加粗] + [本章分析路径预告1句]

**Example**:
> 2024年中国网上零售额达15.52万亿元，移动端贡献了绝对主力流量。**然而，电商行业正在经历十年来最大的格局重塑——抖音电商首次超越京东跻身前三，内容电商从补充渠道升级为主战场。** 本章将通过渠道份额演变追踪、迁移矩阵和效率对比三个维度，剖析这一历史性变局的驱动因素和战略含义。

#### Section Closing Pattern
Each chapter ends with a **bridge paragraph** that connects to the next section or crystallizes the strategic implication:

**Template**:
> [本章核心结论1句] + [对特定利益相关者的行动建议1-2句] + [过渡到下一章的逻辑桥梁1句]

**Example**:
> 渠道迁移的本质不是流量的搬运，而是消费者决策路径的重构——内容电商将"认知→兴趣→购买"压缩到单次观看中完成。品牌应将抖音店播纳入核心渠道策略，而非作为试验性项目运营。这一渠道变局的背后，是两类截然不同的消费者群体在驱动增长——银发经济与Z世代，我们将在下一章展开分析。

#### Data Integration Rules
- **Never list data in isolation** — every number must be embedded in a sentence that explains its significance
- **Bold key figures** inline: "用户规模达**11.16亿**" not "用户规模：11.16亿"
- **Compare, don't just state**: "同比增长**8.4%**，**创近年新高**" not just "同比增长8.4%"
- **Chart references flow naturally**: "如图2所示，时长增速已连续三个季度加速" not "（见图2）时长增速加速"
- **Tables supplement, not replace narrative** — a table may follow a paragraph as supporting evidence, but the insight must already be stated in prose above the table

#### Recommendation Writing Rules
Strategic recommendations should read as **argued conclusions**, not bullet lists:

**Good**:
> 对于品牌而言，渠道策略需要从"货架电商为主、内容电商为辅"彻底转向"内容+货架双轮驱动"。具体来看，抖音店播和货架场景合计已贡献平台GMV的70%以上，这意味着品牌需要建立独立的内容电商团队，将其从市场部的附属职能升级为与电商部平级的核心业务单元。同时，考虑到微信视频号电商即将爆发，提前布局视频号直播将为品牌创造先发优势。

**Bad**:
> 建议：
> - 布局内容电商
> - 投入抖音店播
> - 关注视频号

### Bullet Point Usage (Limited)
Bullets should ONLY be used for:
- Enumerated lists where parallel structure aids readability (e.g., 5 key risks, 3 strategic priorities)
- Data comparison lists within a primarily narrative section
- Executive summary key findings

Bullets should NEVER replace narrative paragraphs in the report body. The default writing mode is **flowing prose with embedded data**, not bullet-point summaries.

## Chart Anchoring and Numbering

- Charts numbered sequentially throughout: 图1, 图2, 图3... 图19
- Each chart has a descriptive title: "图 X: [detailed description of chart's insight]"
- "图 X" in red, description in black
- In-text references use format: "（见图X）"
- Charts placed immediately after or near first text reference
- Typically 1-2 charts per page on chart-heavy pages
- Source attribution below every chart: "资料来源：[Research Partner]；[Consulting Firm]分析"

## Cross-Reference System

- Charts: "见图X" (see Figure X)
- Sections: direct reference by Chinese section title
- External reports: full Chinese title in 《book title》 format with year
- External research: "《X年中国Y报告，系列Z》"
- Footnotes for methodology notes (superscript numbers 1, 2, 3...)

## Header/Footer Rules

### Header (present on all content pages, absent from cover, author page, back cover)
Three-tier header:
- Top center: "[Research Partner] | [Consulting Firm]" in **deep red**
- Horizontal rule below header text
- Below rule: Report subtitle centered in **black**

### Footer
- Page number centered at bottom
- Starts from page 1 (Table of Contents)
- No other footer content

## Content Density Guidelines

- Text-heavy pages: ~800-1200 Chinese characters per page
- Chart pages: 1-2 full-size charts per page with supporting text
- **DO NOT create nearly-empty pages** -- avoid pages with only a single highlight box or one short sentence
- Close-spaced text, generous charts
- Combine thin content with adjacent sections

## What NOT to Include

Based on reference structure, AVOID these elements that were incorrectly added in previous versions:
1. **NO numbered section divider pages** (like "01, 02, 03" with large numbers) -- the reference flows directly
2. **NO methodology description page** -- scope info belongs in executive summary intro or as footnotes
3. **NO appendix section** -- all content is inline
4. **NO numbered subsection system** (like 1.1, 1.2) -- use descriptive headings only
5. **NO nearly-empty pages** -- avoid wasting pages on thin content

## Data Presentation Rules

1. **Lead with the insight, support with data**
2. **Percentages always include the % sign**
3. **Year-over-year comparisons: always state both periods explicitly**
4. **Multi-period trends: use annotated line charts**
5. **Category comparisons: use bar/column charts with all labels**
6. **Market share: use stacked column charts with side tables**
7. **Price trends: use multi-panel mini-charts or scatter plots**

## Source Attribution
- Every chart and data point needs a source
- Standard format: "资料来源：[Research Partner]；[Consulting Firm]分析"
- Adapted format for different data: "资料来源：[Research Source]; [Firm]分析"
- Methodology footnotes placed below chart/table when needed
