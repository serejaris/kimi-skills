# Podcast Script Writer — 播客脚本生成 SOP

根据用户提供的主题、嘉宾信息和节目定位，生成结构完整的播客脚本，涵盖开场白、分段话题、过渡语、预设问题和收尾CTA，所有段落附带时间戳标记。

## Quick Start

1. 用户提供播客主题、嘉宾信息、目标时长等基本信息
2. Agent 按下方 SOP 逐步生成脚本
3. 输出带时间戳的完整播客脚本
4. 用户确认后，可导出为 Markdown 文件

---

## SOP 流程

### Phase 1：需求收集

**目标**：明确播客的基本参数和风格定位。

**步骤**：

1. **收集基础信息**，向用户确认以下字段：

   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 播客名称 | 节目名 | 「创业者说」 |
   | 本期主题 | 本期讨论的核心话题 | AI 如何改变内容创作 |
   | 节目形式 | 独白 / 对谈 / 多人圆桌 | 对谈（主持人 + 1 位嘉宾） |
   | 嘉宾信息 | 姓名、身份、与话题的关联 | 李明，某 AI 公司 CTO |
   | 目标时长 | 节目总时长（分钟） | 45 分钟 |
   | 目标听众 | 核心受众画像 | 25-35 岁互联网从业者 |
   | 风格基调 | 轻松/专业/深度/幽默 | 轻松专业，适度幽默 |
   | 发布平台 | 小宇宙/Apple Podcast/喜马拉雅等 | 小宇宙 + Apple Podcast |

2. **如果用户只提供了主题**，其余字段按以下默认值处理，并告知用户：
   - 节目形式：对谈（主持人 + 嘉宾）
   - 目标时长：30 分钟
   - 风格基调：轻松专业
   - 其余字段标注"待确认"

3. **确认话题子方向**：基于主题，建议 3-5 个可讨论的子话题供用户选择或调整。

---

### Phase 2：脚本结构规划

**目标**：根据时长和话题，规划脚本的时间分配。

**时间分配模板**（以百分比表示，适配任意时长）：

| 段落 | 占比 | 30 分钟示例 | 45 分钟示例 |
|------|------|-------------|-------------|
| 开场白 | 8-10% | 2-3 分钟 | 3-4 分钟 |
| 话题 1 | 20-25% | 6-8 分钟 | 9-11 分钟 |
| 过渡语 1→2 | 2-3% | ~1 分钟 | ~1 分钟 |
| 话题 2 | 20-25% | 6-8 分钟 | 9-11 分钟 |
| 过渡语 2→3 | 2-3% | ~1 分钟 | ~1 分钟 |
| 话题 3 | 20-25% | 6-8 分钟 | 9-11 分钟 |
| 过渡语 3→收尾 | 2-3% | ~1 分钟 | ~1 分钟 |
| 收尾 + CTA | 8-10% | 2-3 分钟 | 3-4 分钟 |

**规则**：
- 话题数量根据时长调整：≤20 分钟 2 个话题，21-40 分钟 3 个话题，>40 分钟 3-4 个话题
- 每个话题内预设 2-4 个问题
- 将时间规划表展示给用户确认后再进入下一步

---

### Phase 3：撰写开场白

**目标**：写出抓住听众的开场白，含节目介绍和本期预告。

**开场白结构**：

```markdown
## 开场白
**[00:00 - {结束时间戳}]**

### 1. Hook（前 15 秒）
> [用一个引人入胜的问题、数据或故事开头，让听众想继续听]

### 2. 节目介绍（15-30 秒）
> 欢迎收听「{播客名称}」，我是你的主持人{主持人名}。
> [一句话介绍节目定位]

### 3. 本期预告（30-60 秒）
> 今天我们聊的话题是：{主题}。
> [简要介绍为什么这个话题值得聊，制造期待感]

### 4. 嘉宾引入（如有，30-60 秒）
> 今天请到的嘉宾是{嘉宾姓名}，{嘉宾一句话介绍}。
> [一个轻松的破冰问题，帮嘉宾自然进入状态]
```

**撰写规则**：
- Hook 必须在前 15 秒内制造好奇心，常用手法：反直觉数据、场景化问题、争议性观点
- 语气贴合风格基调设定
- 嘉宾介绍要突出与本期话题的关联，而非泛泛罗列履历
- 所有文字标注时间戳，格式 `[MM:SS]`

---

### Phase 4：撰写分段话题与预设问题

**目标**：为每个话题段落写出详细的讨论框架和预设问题。

**每个话题段落的结构**：

```markdown
## 话题 N：{话题标题}
**[{开始时间戳} - {结束时间戳}]**

### 话题引入
> [1-2 句话引出这个话题，说明为什么重要/有趣]

### 预设问题

**Q1（破冰/入门）**：[{较轻松、容易回答的问题，帮嘉宾打开话匣子}]
- 预期方向：[嘉宾可能聊到的 2-3 个方向]
- 追问备选：[如果回答太短或偏离，可以用的追问]

**Q2（深入）**：[{推动讨论进入核心观点的问题}]
- 预期方向：[期待嘉宾分享的核心洞察/经验]
- 追问备选：[挖掘细节的追问]

**Q3（实操/案例）**：[{要求嘉宾给出具体案例或方法论}]
- 预期方向：[期待的案例类型]
- 追问备选：[让嘉宾展开细节的追问]

**Q4（争议/前瞻，可选）**：[{有一定争议性或前瞻性的问题}]
- 预期方向：[可能引发的有趣讨论]

### 主持人备注
- [本段需要注意的节奏控制点]
- [如果嘉宾聊嗨了的时间兜底方案]
- [本段的核心 takeaway，确保讨论结束时点到]
```

**问题设计规则**：
- 问题从浅到深排列：破冰 → 深入 → 实操 → 前瞻
- 避免是非题（"你觉得 AI 好不好？"），使用开放式问题
- 每个问题附带"预期方向"和"追问备选"，保证对话不冷场
- 问题之间有逻辑递进关系，不是随机罗列
- 独白型节目：将"问题"替换为"讨论要点"，标注每个要点的核心论述和素材

---

### Phase 5：撰写过渡语

**目标**：写出自然流畅的话题过渡，避免生硬切换。

**过渡语模板**（每段 15-30 秒）：

```markdown
## 过渡语：话题 N → 话题 N+1
**[{时间戳}]**

> [过渡方式：从以下 5 种中选择最自然的]
```

**5 种过渡手法**：

| 手法 | 适用场景 | 示例 |
|------|----------|------|
| 总结承接 | 上一话题有明确结论 | "刚才聊了 X，其实这就引出一个更深的问题……" |
| 对比转折 | 两个话题有对比关系 | "说完了 A 的好处，那另一面呢？" |
| 听众视角 | 需要拉回听众注意力 | "我想很多听众可能会好奇……" |
| 嘉宾线索 | 嘉宾上段提到了相关内容 | "你刚才提到了 Y，这让我想到……" |
| 轻松缓冲 | 上一话题较沉重 | "好，我们换个轻松点的角度……" |

**规则**：
- 相邻过渡语不使用相同手法
- 过渡语要同时回顾上一话题的核心点和预告下一话题
- 保持口语化，避免书面感

---

### Phase 6：撰写收尾与 CTA

**目标**：写出有力的收尾总结和明确的行动号召。

**收尾结构**：

```markdown
## 收尾 + CTA
**[{开始时间戳} - {结束时间戳}]**

### 1. 总结回顾（60-90 秒）
> 今天我们聊了三个话题：
> - {话题1}：核心 takeaway 是……
> - {话题2}：核心 takeaway 是……
> - {话题3}：核心 takeaway 是……

### 2. 嘉宾金句/寄语（30 秒，如有嘉宾）
> 最后想请{嘉宾}用一句话给听众一个建议/寄语？
> [预留嘉宾发言空间]

### 3. CTA — 行动号召（30-60 秒）
> **CTA 1（互动）**：[引导听众参与，如评论区话题、投票]
> 示例："你觉得 AI 会取代内容创作者吗？欢迎在评论区告诉我们。"
>
> **CTA 2（订阅/关注）**：[引导订阅或关注]
> 示例："如果这期内容对你有启发，别忘了点击订阅，下周我们继续聊。"
>
> **CTA 3（资源引导，可选）**：[引导到 show notes、社群等]
> 示例："本期提到的所有资源链接都放在 show notes 里了。"

### 4. 结束语（15 秒）
> [固定的节目结束语，形成品牌记忆]
> 示例："感谢收听「{播客名称}」，我们下期见。"
```

**CTA 设计规则**：
- 每期至少包含 1 个互动 CTA + 1 个订阅 CTA
- CTA 要具体明确，告诉听众"做什么"而非"希望你……"
- 资源引导 CTA 根据实际情况决定是否包含
- CTA 语气自然，避免过度推销感

---

### Phase 7：组装完整脚本

**目标**：将所有段落组装为带时间戳的完整脚本。

**输出模板**：

```markdown
# 播客脚本：{本期主题}

| 项目 | 内容 |
|------|------|
| 播客名称 | {播客名称} |
| 本期主题 | {主题} |
| 节目形式 | {独白/对谈/圆桌} |
| 嘉宾 | {嘉宾信息，如有} |
| 目标时长 | {N} 分钟 |
| 目标听众 | {受众描述} |
| 风格基调 | {风格} |

---

## 时间轴总览

| 时间段 | 段落 | 内容概要 |
|--------|------|----------|
| [00:00-{T1}] | 开场白 | Hook + 节目介绍 + 嘉宾引入 |
| [{T1}-{T2}] | 话题 1 | {话题1标题} |
| [{T2}-{T3}] | 过渡 | {过渡方式} |
| [{T3}-{T4}] | 话题 2 | {话题2标题} |
| [{T4}-{T5}] | 过渡 | {过渡方式} |
| [{T5}-{T6}] | 话题 3 | {话题3标题} |
| [{T6}-{T7}] | 过渡 | {过渡方式} |
| [{T7}-{END}] | 收尾+CTA | 总结 + 行动号召 |

---

## 完整脚本

（依次放入 Phase 3-6 的所有内容，保留时间戳标记）

---

## 附录

### Show Notes 模板

- 嘉宾：{嘉宾信息}
- 时间轴：（从时间轴总览复制）
- 提到的资源/链接：
  - [资源1]
  - [资源2]
- 联系方式：{社交媒体/邮箱}

### 录制前 Checklist
- [ ] 嘉宾已确认录制时间
- [ ] 设备/软件测试完成
- [ ] 脚本已发送给嘉宾预览（可选）
- [ ] 准备好备选话题以防冷场
```

### Phase 8：审校与交付

**自动校验清单**（逐项检查并报告）：

- [ ] 时间戳连续无重叠，总和等于目标时长（±2 分钟容差）
- [ ] 每个话题段落都有 ≥2 个预设问题
- [ ] 每个预设问题都有"预期方向"和"追问备选"
- [ ] 过渡语数量 = 话题数量（含话题到收尾的过渡）
- [ ] 开场白包含 Hook + 节目介绍 + 本期预告
- [ ] 收尾包含 ≥1 个互动 CTA + ≥1 个订阅 CTA
- [ ] 时间戳格式统一为 `[MM:SS]` 或 `[MM:SS - MM:SS]`
- [ ] 语气风格与 Phase 1 设定一致

**交付流程**：

1. 展示完整脚本给用户确认，重点关注：
   - "话题顺序和问题是否合理？"
   - "语气风格是否符合预期？"
   - "CTA 是否需要调整？"

2. 根据用户反馈修订，直到用户确认。

3. 导出选项：
   - 保存为 Markdown 文件
   - 如需其他格式，告知可使用对应 skill 转换

## 参数说明

本 skill 不需要外部参数或 API Key。以下为可选的定制项：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 语言 | 中文 | 脚本输出语言，可切换为英文 |
| 话题深度 | 标准 | 标准（每话题 2-3 问）/ 深度（每话题 3-4 问） |
| CTA 风格 | 温和 | 温和 / 直接 / 幽默 |
| 时间戳精度 | 分钟 | 分钟 [MM:SS] / 秒 [MM:SS]（精确到 15 秒） |

## 常见场景示例

### 场景 1：科技类对谈播客
主持人与嘉宾讨论某个科技趋势，45 分钟，3 个话题，轻松专业风格。

### 场景 2：个人成长独白播客
主播独白分享方法论，20 分钟，2 个话题，深度走心风格。

### 场景 3：商业圆桌播客
3 位嘉宾讨论行业趋势，60 分钟，4 个话题，专业严谨风格。

### 场景 4：英文访谈播客
英文对谈节目，30 分钟，3 个话题，casual & insightful 风格。
