## 可调常量

所有配置常量定义在 `scripts/compare.py` 的顶部：

```python
ALLOWED_EXTENSIONS = {'.mp4', '.mov', '.avi', '.mkv', '.webm'}
MAX_FILE_SIZE_MB = 500          # 每个视频的最大文件大小
FFMPEG_TIMEOUT = 300            # FFmpeg 超时时间（秒）- 5 分钟
FFPROBE_TIMEOUT = 30            # FFprobe 超时时间（秒）- 30 秒
BASE_FRAME_HEIGHT = 800         # 对比帧高度（像素）
FRAME_INTERVAL = 5              # 默认提取间隔（秒）
```

## 文件处理限制

### MAX_FILE_SIZE_MB

**默认值：** 500 MB

**用途：** 防止处理超大视频时内存耗尽。

**何时需要调大：**
- 处理高分辨率或长时间的源视频
- 系统内存充足（16GB+）
- 处理 4K 或 8K 内容

**何时需要调小：**
- 系统内存有限
- 在低配机器上处理
- 同时批量处理多个视频

**影响：** 不影响输出画质，仅决定哪些文件可以被处理。

### FFMPEG_TIMEOUT

**默认值：** 300 秒（5 分钟）

**用途：** 防止 FFmpeg 操作无限挂起。

**何时需要调大：**
- 处理很长的视频（> 1 小时）
- 提取大量帧（`--interval` 值很小）
- 存储速度慢（网络驱动器、外置硬盘）
- 高分辨率视频（4K、8K）

**推荐值：**
- 短视频（< 10 分钟）：120 秒
- 中等视频（10-60 分钟）：300 秒（默认）
- 长视频（> 60 分钟）：600-900 秒

**影响：** 超时会导致操作失败；不影响输出画质。

### FFPROBE_TIMEOUT

**默认值：** 30 秒

**用途：** 防止元数据提取挂起。

**何时需要调大：**
- 通过慢速网络访问视频
- 处理编码结构复杂的文件
- 损坏或格式异常的视频文件

**典型行为：** 元数据提取通常在 5 秒内完成；耗时更久通常说明文件有问题。

**影响：** 超时会导致操作失败；不影响输出画质。

## 帧提取设置

### BASE_FRAME_HEIGHT

**默认值：** 800 像素

**用途：** 统一帧尺寸，便于并排对比。

**何时需要调大：**
- 对比高分辨率视频（4K、8K）
- 分析细微细节或压缩伪影
- 在大屏幕上查看报告

**何时需要调小：**
- 加快处理速度、减小 HTML 文件大小
- 在手机或小屏幕上查看报告
- 带宽有限，需要分享报告

**推荐值：**
- 移动端/低带宽：480-600 像素
- 桌面端查看：800 像素（默认）
- 高细节分析：1080-1440 像素
- 4K/8K 分析：2160+ 像素

**影响：** 值越大，HTML 文件越大、处理时间越长，但保留的细节越多。

### FRAME_INTERVAL

**默认值：** 5 秒

**用途：** 控制帧提取频率。

**何时需要调小（提取更多帧）：**
- 分析快速运动的内容
- 需要详细的时序分析
- 短视频，更多采样有助于分析

**何时需要调大（提取更少帧）：**
- 长视频，减少处理时间
- 缩小 HTML 输出文件
- 概览分析（快速画质检查）

**推荐值：**
- 快速运动/高细节：1-3 秒
- 标准分析：5 秒（默认）
- 长视频内容：10-15 秒
- 快速概览：30-60 秒

**影响：**
- 间隔越小：帧数越多、HTML 越大、处理越慢、分析越全面
- 间隔越大：帧数越少、HTML 越小、处理越快、可能遗漏瞬时伪影

## 配置影响

### 处理时间

处理时间主要受以下因素影响：
1. 视频时长
2. `FRAME_INTERVAL`（越小 = 帧越多 = 处理越慢）
3. `BASE_FRAME_HEIGHT`（越高 = 像素越多 = 处理越慢）
4. 系统 CPU/存储速度

**典型处理时间：**
- 5 分钟视频，5 秒间隔，800px 高度：约 45-90 秒
- 30 分钟视频，5 秒间隔，800px 高度：约 3-5 分钟
- 60 分钟视频，10 秒间隔，800px 高度：约 4-7 分钟

### HTML 输出大小

HTML 文件大小主要受以下因素影响：
1. 提取的帧数
2. `BASE_FRAME_HEIGHT`（越高 = base64 编码图片越大）
3. 视频复杂度（细节丰富的帧压缩效率更低）

**典型 HTML 大小：**
- 5 分钟视频，5 秒间隔，800px：5-10 MB
- 30 分钟视频，5 秒间隔，800px：20-40 MB
- 60 分钟视频，10 秒间隔，800px：30-50 MB

### 画质与性能权衡

**高画质配置（细致分析）：**
```python
MAX_FILE_SIZE_MB = 2000
FFMPEG_TIMEOUT = 900
BASE_FRAME_HEIGHT = 1440
FRAME_INTERVAL = 2
```
适用场景：精细画质分析、存档对比、专业编码器评估

**均衡配置（默认）：**
```python
MAX_FILE_SIZE_MB = 500
FFMPEG_TIMEOUT = 300
BASE_FRAME_HEIGHT = 800
FRAME_INTERVAL = 5
```
适用场景：常规压缩分析、桌面端查看

**快速处理配置（快速概览）：**
```python
MAX_FILE_SIZE_MB = 500
FFMPEG_TIMEOUT = 180
BASE_FRAME_HEIGHT = 600
FRAME_INTERVAL = 10
```
适用场景：批量处理、快速画质检查、移动端查看

## 允许的文件扩展名

**默认值：** `{'.mp4', '.mov', '.avi', '.mkv', '.webm'}`

**用途：** 限制输入为已知的视频格式。

**何时需要修改：**
- 添加对其他容器格式的支持（如 `.flv`、`.m4v`、`.wmv`）
- 为了工作流标准化而限制特定格式

**注意：** 添加扩展名不保证兼容性；FFmpeg 必须支持对应的编解码器/容器。

## 安全注意事项

**不要修改：**
- 路径校验逻辑
- 命令执行方式（必须避免 `shell=True`）
- 异常处理模式

**可以安全修改：**
- 数值限制（文件大小、超时时间、尺寸）
- 允许的文件扩展名（添加 FFmpeg 支持的格式）
- 输出格式偏好

**不安全的修改：**
- 移除路径清理
- 绕过文件验证
- 启用 shell 命令插值
- 禁用资源限制
