# Исключённое при выгрузке

Исходная директория `/app/.agents/skills`: **128 844 файлов, 1.7 ГБ**. В подготовленный дамп вошли **2 574 файла (~38 МБ)** — сами скиллы без окружения.

## Что исключено

| Категория | Что внутри |
|---|---|
| `node_modules/` | Зависимости шаблонов webapp-building / backend-building (TypeScript, esbuild, rolldown и т.д.) — восстанавливаются через `npm install` |
| Файлы > 2 МБ | Бинарники: `xlsx/scripts/Xlsx` (74 МБ), `tsc` (23 МБ), `rolldown-binding` (19 МБ), WIR-движок `_core.cpython-312-x86_64-linux-gnu.so` (15 МБ), zip-архивы шаблонов, esbuild (11 МБ) |
| `__pycache__/`, `*.pyc`, `.DS_Store`, `.git/` | Служебные файлы |

## Почему не всё в репо

Прямой пуш через GitHub API пропускает контент только текстом через контекст агента — 38 МБ туда не помещаются. Поэтому:

1. Сейчас в репо: каталог всех 274 скиллов + полные `SKILL.md` 16 скиллов, непосредственно доступных агенту.
2. Полный дамп (2 574 файла, включая все `SKILL.md` остальных 258 скиллов, скрипты, референсы, шаблоны) упакован в `kimi-skills.tar.gz` и залит на временное хранилище.
3. Для заливки дампа подготовлен workflow GitHub Actions: он скачивает архив, распаковывает и коммитит всё одним прогоном с чистой историей.

**Блокер:** у OAuth-токена GitHub-интеграции нет scope `workflow` — GitHub отклоняет создание файлов в `.github/workflows/` через API (проверено, обычные файлы пушатся нормально).

**Разблокировка:** переподключить плагин GitHub в Kimi с расширенными правами (workflow) — после этого один пуш workflow-файла дольёт весь дамп автоматически.
